# Gemini词汇清理清单（chap1-11，第一轮）

## 说明

- 范围：`chapters/chap1.tex` 到 `chapters/chap11.tex`
- 目标：清理“看起来高级、实际上没有帮助”的措辞，尽量改回朴素、初等、直接的数学表述。
- 本轮结论：`泛函`、`萃取` 这两个词在 `chap1-11` 中未检出；但同类问题很多，主要集中在 `chap3.tex`、`chap5.tex`、`chap11.tex`。
- 特别说明：`chap11.tex` 里有大量 `%` 注释。注释不影响排版，但如果要统一源码风格，也建议一起清。

## 判定标准

- 能直接说“设函数”“作差”“求导”“分离变量”的，不要硬说成“核心结构”“解耦”“参数化”“构型”。
- 能直接写“前几项一致”“高阶项相同”的，不要硬说成“匹配”“拟合”“高阶渐近匹配”。
- 能直接讲“附近的变化”“最高次项”“主要部分”的，不要硬说成“局部性态”“发散主部”“核心部分”。
- 术语如果确实是标准数学术语，可以保留；如果只是为了显得高级，就换掉。

## 高频替换词典

- `本质` / `本质上` -> `其实`、`实际上`，很多地方可以直接删掉
- `核心` -> `关键`、`主要`，或直接点名具体对象
- `结构` -> `形式`、`式子`、`表达式`
- `图景` -> `直观理解`、`几何解释`
- `降维` -> `降次`、`化成一元`、`少掉一个变量`
- `耦合` / `解耦` -> `变量缠在一起` / `分离变量`
- `参数化` -> `引入参数`、`设参数`
- `匹配` -> `一致`、`前几项相同`
- `拟合` -> `逼近`、`构造一个有理式使前几项一致`
- `析出` -> `提出`
- `剥离` -> `化掉`、`提出`
- `局部性态` -> `附近的变化`
- `构型` -> `形式`
- `过渡函数` -> `中间函数`
- `主结构` -> `主要部分`
- `发散主部` -> `最高阶项`
- `完整过程` -> `整个推导`

### chap3.tex

1. `chap3.tex:135`
   - 原句摘录：`引入微积分中的“微分算子平移定理”……使原本含有较高次指数的项平移为新的常数项多项式`
   - 问题：`微分算子平移定理`、`平移`、`指数基数` 一套说法过重，不像本书前后文的语言层级。
   - 建议替换：改成“把每一项反复求导，并同时乘上 `e^{-x}`，这样高次指数项会逐步降下来，最后只剩一个多项式需要判断符号。”
   - 优先级：高

2. `chap3.tex:141`
   - 原句摘录：`复合微分算子`、`泰勒截断`
   - 问题：术语味过浓，容易把本来直接的求导计算写成抽象算子语言。
   - 建议替换：直接写“把 `(D+1)^5 (D+2)^5 (D+3)^4 (D+4)^3` 展开到一次导数项即可，因为 `P_4(x)` 只有一次。”
   - 优先级：高

3. `chap3.tex:161`
   - 原句摘录：`降阶逻辑`、`导数剥离`、`复合镜像微分算子`、`算子截断`、`代数算子推演`
   - 问题：这一串几乎都是“说得很大”的词，但实际内容只是多次求导与整理。
   - 建议替换：统一改成“继续按前面的办法多次求导”“把高次项一层层降下来”“整理后得到下面这个二次多项式”。
   - 优先级：高

4. `chap3.tex:171`
   - 原句摘录：`核心部分`、`唯一决定符号的因子`
   - 问题：语气过满，可以更平实。
   - 建议替换：改成“因为 `D(x)>0`，`M(x)<0`，所以符号最后只由 `(e^x-1)` 决定。”
   - 优先级：中

5. `chap3.tex:331`
   - 原句摘录：`超越耦合`、`对数平均比值参数化`
   - 问题：`耦合`、`参数化` 都偏 AI 化；这里其实就是设比值参数。
   - 建议替换：改成“为把 `x,y` 分开，设 `t=\frac{x}{y}``。”
   - 优先级：高

6. `chap3.tex:333`
   - 原句摘录：`为进一步利用偶函数的对称解析性质`
   - 问题：`解析性质` 在这里没必要。
   - 建议替换：改成“再设 `v=\frac{\ln t}{2}`，这样式子会变成偶函数，后面更容易处理。”
   - 优先级：中

7. `chap3.tex:342`
   - 原句摘录：`放缩链条`、`中间过渡函数`、`渐近特征`
   - 问题：说法太像模型说明书，不像题解。
   - 建议替换：改成“为了夹在两边之间，构造一个有理式作为中间函数，同时照顾 `v\to0` 和 `v\to+\infty` 时的行为。”
   - 优先级：高

8. `chap3.tex:345`
   - 原句摘录：`综合微观极值点与宏观无穷远处的渐近阶数分析`、`两点帕德逼近`、`叠加`、`原生变量`
   - 问题：整句都有明显 Gemini 腔，信息其实可以很朴素地说清。
   - 建议替换：改成“分别比较 `v\to0` 和 `v\to+\infty` 时的前几项，构造 `P(v)=\frac{2v^4}{4v^2+15}`。再把它加到 `R(v)` 上，得到中间函数 `M(v)`，最后换回 `x,y`。”
   - 优先级：高

9. `chap3.tex:344`
   - 原句摘录：`发散主部`
   - 问题：不如直接说“最高阶项”。
   - 建议替换：`最高阶项`
   - 优先级：中

10. `chap3.tex:874`
   - 原句摘录：`等价转化的核心不等式成立`
   - 问题：`核心` 可删。
   - 建议替换：改成“于是上面的不等式成立”或“于是所需不等式成立”。
   - 优先级：低

### chap5.tex

1. `chap5.tex:239`
   - 原句摘录：`以 e^x-x 为核心变量`、`有效分离超越变量并降低不等式的分析复杂度`
   - 问题：`核心变量`、`超越变量`、`分析复杂度` 都很像 AI 技术文。
   - 建议替换：改成“注意右边只和 `e^x-x` 有关，所以设一个中间函数 `f(x)` 来夹住两边。这样两边的 `e^x` 可以消去，式子也更好处理。”

   - 建议替换：改成“这样在 `0` 附近就能压住目标函数。又因为偶函数展开式里没有奇次项，所以可以考虑用 `\cosh` 来构造。”
   - 优先级：高

5. `chap5.tex:247`
   - 原句摘录：`局部高阶匹配条件`

7. `chap5.tex:278`

   - 优先级：高

9. `chap5.tex:389`

   - 问题：这一整段是高风险区，术语密度太大。
   - 建议替换：改成“这个有理函数是按 `x=1` 附近的展开式构造出来的。我们让它在 `x=1` 处的函数值和前几阶导数与原函数一致，于是能解出系数。这样误差里自然会带出 `(x-1)^3` 因子，后面判断单调性就方便了。”
