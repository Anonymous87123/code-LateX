# MD/TEX 学术文风精细化分析执行计划书

## 1. 任务目标与不可替代的主次关系

本项目的最终目标不是继续扩大聊天口头禅统计，而是从历史 MD/TEX 正式成文中，人工拆解可复核的 AI 学术文风问题，并把结论转化为可执行的论文改写规则与独立 Skill。

样本权重固定为：

| 层级 | 权重 | 样本 | 用途 |
|---|---:|---|---|
| 核心样本 | 90% | 可确认或高可信的 GPT 生成 MD/TEX 正式稿 | 定义学术文风问题、提供原文案例、形成规则 |
| 辅助样本 | 10% | `.codex` assistant 聊天和生成上下文 | 确认任务场景、迭代原因与作者归属，不生成学术规则 |
| 审计对照 | 不计权 | 第三方 README、依赖文档、技能模板、来源不明文件 | 检查来源混杂，禁止归因给 GPT |

聊天中的“我先、已完成、下一步”等表达只保留在聊天风格档案，不得进入论文去 AI 规则的证据链。

## 2. 全域归集与来源分级

### 2.1 归集范围

1. 递归读取 `C:\Users\Lenovo\.codex` 中的 JSONL、MD 及缓存记录。
2. 递归扫描 `D:\` 全部 `.md` 与 `.tex`，包括子目录与可访问的隐藏目录。
3. 纳入用户明确指定的 GPT 样本：
   - `D:\code LateX\elegantbook\CET6\cet6` 中除 `cet6.tex` 外的 TeX；
   - 用户提供的微信文件 `main.tex`；
   - 用户在对话中明确说明为 GPT 生成的其他 TeX。
4. 无法解码的文件只进入清单并标记 `undecodable`，不阻塞进度，也不凭乱码推断文风。

### 2.2 来源证据等级

| 级别 | 判定 | 可否用于核心案例 |
|---|---|---|
| A1 | 用户明确说明该文件由 GPT 生成 | 可以 |
| A2 | assistant 可见输出明确声明写入该路径，且文件内容/时间可对应 | 可以 |
| B | 日志、文件历史与内容高度一致，但缺少单次写入闭环 | 可用于模式观察，案例须标注“高可信推定” |
| C | 仅在日志、工具输出或提示中出现路径 | 不作作者归因 |
| D | 技能、依赖、第三方 README、论文原文或翻译来源 | 只作混杂源/对照 |

路径提及不等于生成证据。重复哈希、版本副本和同一模板族按一个写作家族计权，防止同义复制放大结论。

### 2.3 机器工具的职责边界

脚本只执行：文件清单、哈希、格式/编码识别、脱敏、去代码/公式的正文视图、段落与句子索引。脚本不得自动输出“AI 味重”“人类化”“严谨”等文风结论。所有文风结论必须由人工完整阅读文档、章节和上下文后写出。

## 3. 双语境独立分析体系

### 3.1 聊天口语风格体系（留存档案）

只记录过程播报、完成态、状态限定、建议尾句、工程词汇与 commentary/final 的结构差异。该体系用于解释正式稿为什么出现项目管理语气，不参与学术论文规则提炼。

### 3.2 学术写作风格体系（核心）

每份核心文稿作为最小阅读单元，先确定文档任务，再逐章检查：

1. 词汇：抽象万能词、无证据形容词、元写作控制词、术语漂移、英语夹杂。
2. 句式：二元对照、排比、定义句、限定句、让步句、长句负载、主语选择。
3. 逻辑：问题如何导出方法、证据如何支持主张、是否存在伪递进或预制闭环。
4. 衔接：段首路标、段末预告、连接词依赖、跨段因果与回指对象。
5. 段落：单段功能、主次比例、论据位置、是否每段机械等长。
6. 论证：主张强度、证据等级、反例、适用域、局限与外推。
7. 结构：摘要/引言/方法/实验/讨论的职责，标题密度与清单化倾向。
8. 节奏：长短句变化、重点段与过渡段的篇幅差、是否过度工整。
9. 话语角色：论文作者、项目经理、审计员、教师、产品介绍者之间的切换。
10. 模板复用：固定背景、固定结尾、跨文档公共句、同一论证骨架的迁移。

## 4. AI 学术文风判定指标库

### 4.1 词汇指标

- `重要、显著、有效、全面、系统、深入、充分、稳健`等评价词是否有相邻证据。
- `赋能、闭环、落地、抓手、主线、口径、范式`是否替代了可观察动作。
- `本文旨在、具有重要意义、综上所述、值得注意的是`是否承担真实信息。
- 同一概念是否在“机制/框架/体系/路径/模式”之间无定义切换。

### 4.2 句式指标

- `不是 A，而是 B`是否制造了不存在的二选一。
- `一方面/另一方面`、`首先/其次/最后`是否只是均匀分箱。
- 三项排比是否为逻辑需要，还是为了句面完整。
- 长句是否同时塞入背景、方法、结果、价值与展望。
- 每段是否以总结句封口，导致没有自然的论证留白。

### 4.3 逻辑与论证指标

- 方法是否由前文具体缺口推出，还是突然出现“为解决上述问题”。
- 结果是否支持主张的强度，是否从局部样本外推为普遍规律。
- 是否用“边界声明”替代真正的实验、对照或统计检验。
- 是否先预设完整框架，再把材料机械填入所有栏目。
- 是否重复结论来制造确定感，而非增加新证据。

### 4.4 段落与节奏指标

- 相邻段长度、句数、句法是否过度整齐。
- 标题是否比正文信息更密，形成“目录代替论证”。
- 每段是否固定为“主题句-三点解释-总结句”。
- 关键发现是否得到更长、更具体的证据篇幅；次要内容是否主动收短。
- 是否允许读者从数据或案例自行得出部分推论，而非每处都由作者代替解释。

## 5. 人类自然学术写作对照标准

所谓“人类化”不等于口语化、删专业词或故意写乱，而是：

1. 用词精度高于抽象覆盖率：一个可测变量胜过三个万能名词。
2. 句式服从论证负担：定义可短，机制解释可长，转折不追求对称。
3. 逻辑有重点倾斜：关键证据展开，常识背景压缩。
4. 段落长度由任务决定：证据段、解释段和过渡段不必整齐。
5. 主张允许有边界，但边界必须与样本、方法或实验条件绑定。
6. 作者可以作判断，但判断应说明依据，不以中庸平衡冒充客观。
7. 适度留白：已被图表直接显示的内容不逐项复述，避免过度讲解。

## 6. 人工阅读与记录格式

每份核心文档建立阅读卡，至少记录：

```text
文档路径 / 来源等级 / 文档任务 / 版本族
章节结构 / 主论证链 / 证据类型 / 读者角色
高价值写法 / AI 模板痕迹 / 重复结构 / 边界问题
原文片段 / 片段位置 / 为什么有问题 / 人类化改写
可形成的强制规则 / 不能外推的结论
```

每个最终文风条目必须包含五件套：

1. 问题定义；
2. 核心 MD/TEX 原文劣例；
3. 成因与判断依据；
4. 人类化改写范例；
5. 可执行的生成约束和自检问题。

## 7. 文体家族与重复控制

初步人工阅读已确认至少以下家族，后续按家族而非简单文件数组织：

- 学术论文正文与 TeX 教材；
- 论文重写施工图/叙事控制文档；
- 结果边界与方法自辩文档；
- 实验总控与证据门禁计划；
- 文献精读到项目决策的翻译器；
- 项目交接与可审计治理报告；
- 工程运行手册；
- 教材/应试反陷阱讲义；
- 审计台账与结构化核对表；
- 产品/竞赛展示稿；
- 第三方模板与技能说明（仅作混杂对照）。

同一家族中完全相同或高度复用的文档，只能证明“模板稳定”，不能按文件数重复证明“风格普遍”。

## 8. 最终 Skill 框架

独立 Skill 手册固定包含：适用范围、强制规则、分阶段工作流、AI 痕迹清单、逐项改写规则、文档级检查、段落级检查、句子级检查、证据门禁、失败示例、输出格式、自检清单。

规则强度分为：

- `MUST`：违反即视为不合格，如无证据的结果外推、伪造引用、术语未定义。
- `SHOULD`：默认执行，如压缩元话语、打破机械三分法。
- `MAY`：按文体选择，如第一人称、章节预告、项目管理标签。

## 9. 验收标准

- MD 与 TEX 均有全域清单，乱码/空文件/重复版本有状态记录。
- 核心报告的学术写作篇幅和案例量显著高于聊天篇幅。
- 每个学术文风结论均来自完整文档阅读，不以聊天词频作证。
- 每项主要 AI 痕迹至少有一个真实 MD/TEX 片段和一个可落地改写。
- 独立 Skill 可以直接用于生成、审校和重写论文，而非停留在描述层。
- 第三方材料、技能模板与仅被路径引用的文档不被错误归因为 GPT 原创。
- 所有公开到报告的片段经过敏感信息检查，不含密钥、身份信息或私密路径。
