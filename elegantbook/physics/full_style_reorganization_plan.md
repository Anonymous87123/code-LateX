# 全域 GPT 中文风格报告重整专项计划

## 1. 输入与边界

本次重整只使用 `full_style_corpus_2026-07-10` 的可复跑产物，不再凭人工记忆补写结论：

| 输入 | 数量 | 用途 | 边界 |
|---|---:|---|---|
| C 盘 `.codex` JSONL | 1,194 文件 | 重建会话、模型/人格/阶段与 MD 路径关联 | assistant 的可见 `output_text` 才进入聊天风格语料。 |
| assistant 中文聊天消息 | 117,755 条 | 聊天语言风格主样本 | user、system、developer、reasoning 和工具输出不作为聊天文风。 |
| 全域 MD 文件 | 7,784 文件 | 写作语料全量清单 | 不可读/空文件保留审计；不会悄悄省略。 |
| 可读 MD | 7,779 文件 | 全语种结构、来源与版本统计 | 大量为技能、依赖和英文文档，不可直接当作 GPT 中文原创。 |
| 含中文 MD 段落 | 24,520 段 | 中文写作风格主样本 | 是全语种 MD 的可比中文子语料，不是选择性排除。 |
| MD 路径关联 | 324,045 条 | 生成路径、引用路径、日志关联审计 | 分为明确 assistant 输出与一般日志引用，不将引用等同生成。 |

所有最终数字均链接或可回查到以下文件：`corpus_audit.json`、`jsonl_manifest.csv`、`markdown_manifest.csv`、`md_provenance.jsonl`、`feature_by_group.csv`、`document_profiles.csv`、`phrase_candidates.csv`、`sentence_starts.csv`。

## 2. 重整目标

生成新的 `gpt_chinese_style_report_restructured.md`，不再以“某次任务/某个文件”的时间线叙述，而是严格采用两大一级板块：

1. `聊天语言风格`
2. `写作语言风格`

随后设置一个只做比较、不替代前两板块的 `跨语境连续性与切换` 章节。旧的 `gpt_chinese_style_report.md` 和 `gpt_chinese_style_report_detailed.md` 保留为历史报告与专题证据，不删除、不覆盖。

## 3. 最终文档框架

### 0. 方法与样本审计

- 全域扫描范围、快照时间、文件状态、读取错误、重复版本、脱敏规则。
- C 盘会话中 `.md` 路径的溯源规则：`明确 assistant 输出`、`日志引用`、`未能证实`。
- D 盘兜底扫描为何被触发，以及 2,018 个 D 盘 MD 的纳入状态。
- 全语种 MD 与中文写作子语料的双口径说明；第三方依赖和技能文档不被误归因。

### 1. 聊天语言风格

每个二级章节都固定包含：定义、量化指标、句式/用词、功能解释、阶段差异、优点、AI 痕迹、边界和脱敏证据。

1. `基础特征`：消息长度、中文比例、标点、阶段规模与模型/人格分布。
2. `句式特征`：`我先/我会`、`先……再……`、`不是……而是……`、条件行动句、结论开头和邀请收尾。
3. `用词特征`：完成态、状态限定、行动词、证据词、风险词、结论词、建议词和学术/工程词簇。
4. `逻辑特征`：动作承诺 -> 发现 -> 确认/验证 -> 边界 -> 下一步的过程链。
5. `语气情感`：协作性、确定性与保守性如何并存；低寒暄、低情绪、低感叹的工程沟通气质。
6. `结构范式`：`commentary` 的现场播报与 `final_answer` 的交付报告；按模型/人格展示但不对小样本强归因。
7. `表达习惯与专属特点`：文件/测试/验证导向、会话内重复、模板化收尾。
8. `量化对照`：逐项列出 commentary 与 final_answer 的覆盖率、Wilson 区间和样本数。

### 2. 写作语言风格

同样使用基础、句式、用词、逻辑、语气、结构、表达习惯、专属特点八层，但每层同时区分来源：

1. `全域文档构成`：Codex MD、D 盘 MD、可读/空/不可解码、重复文档、英文/中文分层。
2. `中文写作基础特征`：24,520 个含中文段落的长度、段落密度、标题、列表和文档画像。
3. `句式特征`：无主语论证、`本文/本报告`、`不是……而是……`、条件-行动、范围限定和结论句。
4. `用词特征`：学术词、证据词、风险/局限词、建议词、模型/数据/方法词和工程管理词。
5. `逻辑特征`：问题重述、范围/口径、机制解释、证据链、反直觉修正、建议与闭环。
6. `语气情感`：正式、低情绪、低第一人称；本地报告、项目文档和其他中文文档的差异。
7. `结构范式`：报告体、论文体、模板体、项目说明体、技能/依赖说明体的来源标签与不可混淆边界。
8. `逐文档画像索引`：按 `document_profiles.csv` 提供最大中文文档、最高学术词比、最高对照句比和重复族谱的可追溯索引。

### 3. 跨语境连续性与切换

- 连续性：步骤化、边界意识、证据/验证、结论压缩、`不是……而是……`。
- 切换：聊天使用代理第一人称和过程动词；正式写作使用无主语、`本文/本报告`、章节化论证和读者操作化。
- 不将“聊天高频”直接等同于“正式写作高频”；所有对比使用各自语料分母。
- 说明全域写作样本混合来源造成的限制，单列本地报告与可识别 GPT 成稿专题。

### 4. 结论、仿写与去 AI 化

- 给出聊天体、报告体、论文体、模板体的可复核句式模板。
- 区分“保留的专业特征”与“应压缩的模板痕迹”。
- 每项建议明确对应的量化特征，避免空泛地说“更自然”。

## 4. 内容填充规则

1. 每个强结论必须同时给：样本量、覆盖率/计数、比较分母、来源层级和至少一条脱敏 evidence ID。
2. 全语种 MD 只用于全域构成和来源审计；中文语言风格使用含中文段落子语料，并在每一表注明分母。
3. 只把 `explicit_assistant_output` 级路径称为“日志明确指向 assistant 输出”；其余仅称“日志引用”。
4. 完全相同哈希的 MD 用版本族谱呈现，报告频率默认按文档发生次数和去重后文档各给一列。
5. 不把 `node_modules`、技能说明、产品文档的文风归因给 GPT；它们保留为对照来源和审计对象。
6. 不引用未脱敏正文；所有例句都来自已脱敏的 chat/paragraph/sentence 产物。
7. 对模型/人格差异只在每组聊天消息不少于 500 条时写“明显”；其余只列数据。

## 5. 成文后的验收

- 两大一级板块和其八类细分维度均存在，且不存在将写作/聊天结论混淆的章节。
- 报告中的总数能回加到审计文件：1,194 JSONL、7,784 MD、117,755 聊天消息、24,520 中文写作段落。
- 324,045 条路径关联被说明为证据记录，而非自动等同于生成记录。
- 逐文档画像、路径清单、阶段特征、来源特征和脱敏证据均有可点击或明确的本地路径。
- 运行密钥、Token、私钥、邮箱、手机号、IP 和用户目录路径的模式扫描必须为 clean。
