# 学术文风报告结构化重整专项规划方案

## 1. 重整理由

现有 `gpt_chinese_style_report_restructured.md` 用大量聊天统计开篇，正式写作部分只有总体频率和四种粗分类，无法承载“以 MD/TEX 学术稿为核心、聊天为辅”的任务。重整必须改变证据中心，而不只是增加几十行补充说明。

## 2. 最终产物

1. `gpt_chinese_style_report_restructured.md`：双板块总报告；聊天精简留档，学术写作深度展开。
2. `academic_writing_deai_skill.md`：可独立使用的去 AI 学术写作 Skill 手册。
3. `manual_reading_ledger.md`：人工精读文档、来源等级、文体家族、证据片段与结论台账。
4. `academic_style_analysis_execution_plan.md`：前置分析规则。
5. 本文件：报告重整和案例排布规则。

## 3. 报告权重与结构

### 板块一：模型聊天语言风格库（约 10%）

只保留：过程播报、完成态、验证意识、边界限定、常见口头禅、commentary/final 差异，以及这些特征为什么不能直接迁入论文。

### 板块二：模型学术写作语言风格库（约 90%）

按以下顺序展开：

1. 样本归因与人工阅读账本；
2. 学术/专业文体家族地图；
3. 词汇层 AI 痕迹；
4. 句式层 AI 痕迹；
5. 衔接与逻辑层 AI 痕迹；
6. 段落与章节结构 AI 痕迹；
7. 论证、证据与边界问题；
8. 行文节奏与重点分配；
9. 不同文体的风格切换；
10. 高频模板族与重复控制；
11. 人类化改写案例库；
12. 可执行规则摘要与 Skill 链接；
13. 方法局限与不可归因来源。

## 4. 每个分析小节的固定格式

```text
问题定义
人工观察（说明来自哪些完整文档）
原文劣例（MD/TEX，注明文件与章节）
为什么显得像 AI / 何时并非问题
人类化改写
MUST / SHOULD / MAY 规则
自检问题
```

这套格式避免“列了口头禅却不知道怎么改”的旧问题。

## 5. 案例排布

- 每个大类至少两种文体，防止把项目报告习惯误当作论文普遍规律。
- 优先使用 A1/A2 来源；B 级材料标注“高可信推定”，C/D 不作核心劣例。
- 引文长度以能理解上下文为准，不截取半句制造结论。
- 同一原文最多承担两个高度相关规则，避免反复消费一条例句。
- 对有价值的 AI 写法也要保留：边界校准、证据分层、方法责任明确不能被一概删除。

## 6. Skill 输出映射

报告中的每一项问题都映射到 Skill 的唯一规则 ID，例如：

| 报告问题 | Skill 规则 |
|---|---|
| 无证据评价词 | `LEX-01` |
| 伪二元对照 | `SYN-02` |
| 机械三分法 | `SYN-04` |
| 方法突然出现 | `LOG-01` |
| 结论复述过密 | `PAR-03` |
| 边界声明替代验证 | `EVD-03` |
| 标题密度超过论证密度 | `DOC-02` |
| 全文节奏过度均匀 | `RHY-01` |

## 7. 验收门槛

- 报告不是“聊天分析 + 写作摘要”，而是“学术写作主报告 + 聊天附档”。
- 至少覆盖词汇、句式、逻辑、衔接、段落、论证、结构、节奏八个维度。
- 至少覆盖论文、实验计划、结果边界、文献精读、工程手册、教材讲义六个文体家族。
- 每个核心维度均有真实案例、改写、规则和自检问题。
- Skill 手册不依赖读者回看聊天日志即可独立执行。
- 来源边界、重复模板和不可解码文件均在方法限制中公开说明。
