# 全域学术样本场景分类报告

生成日期：2026-07-10  
分类基线：9,423 份可读 MD/TEX；207 份人工精读核心候选  
场景体系：课堂学习与课程笔记、数学建模与工程仿真、高端科研与期刊论文

## 1. 结论

原有统一 `deai-academic-writing` Skill 的事实、来源、数学正确性底线可以保留，但文风标准不能继续统一执行。课堂笔记需要允许教学拆步和学习复盘；建模论文需要允许有来源、有误差说明的工程近似；高端科研稿需要更强的创新主张、命题证据、论证取舍和期刊结构约束。

本次分类没有把 9,423 个文件强行伪装成学术正文。全域 manifest 包含插件文档、依赖 README、Skill 指令、会话报告、重复备份、OCR 原文和结构化实验记录。所有文件均获得分类决定，但非目标材料进入审计排除层，不参与三场景风格归纳。

## 5. 场景一：课堂学习与课程笔记

### 5.1 来源家族

- `elegantbook`、`beautybook` 教材；
- 简单导数、圆锥曲线、物理等章节 TeX；
- CET-6 作文、听力和 Section B 策略；
- 历年试题、答案重建、OCR 与解答审计；
- 课程习题、章节知识复盘和基础公式推导。

### 5.2 文体合同

目标不是模拟期刊论文，而是帮助学习者理解、复查和再次使用知识。允许：

- 适量步骤拆解；
- 直白解释“为什么选这个方法”；
- 一个学习回顾或易错点；
- 教学需要的局部重复；
- 对基础对象给出比科研论文更多的解释。

仍然禁止：错误公式、遗漏定义域、假定题面来源、伪术语、用“显然”掩盖关键证明、把启发式写成定理。

### 5.3 人工观察到的专属 AI 痕迹

- 章节、节、例题、解答、坑点、口诀六层重复同一路线；
- 每次代入都配一句旁白，决定性构造却只写“化简可得”；
- `入口、路线、底座、母式、送回、收口`在每层复现；
- 为显得高级而引入 `泛函、同构、虚圆、系统不变量`；
- 过度工整的“定义-解释-例题-总结”让学习笔记失去真实复盘重点；
- 教练/游戏/表演词掩盖方法强度和适用条件。

### 5.4 应保留的自然教学特征

- 先说目标，再展示关键一步；
- 例行运算可压缩，但初学者容易错的符号和域要展开；
- 一个主例 + 一个反例/边界；
- 允许短句提示学习者注意对象变化；
- 回顾只记录本节真正新增的判断，不重放全文。

## 6. 场景二：数学建模与工程仿真

### 6.1 来源家族

- 广州高校数学建模邀请赛长江渔业论文；
- LSGO 算法 README、实验协议、指标筛选、API 和实验记录；
- Probet 可视化与闭环审计资产；
- 工程项目 README、产品问卷和验收文档。

### 6.2 文体合同

目标是让模型、数据、代码和决策可对应。允许合理工程近似，但必须写清：

- 近似对象和理由；
- 参数来自观测、校准、文献、假设还是人工设定；
- 误差、敏感性和未检验维度；
- 论文方程、代码实现、参数表和结果表之间的一致性；
- 工程上选择某方案而非另一方案的可复核取舍。

### 6.3 人工观察到的专属 AI 痕迹

- 单个约数锚点输出三至五位小数，制造测量精度错觉；
- 把校准命中写成模型准确，把局地方向一致写成外部验证；
- 只扰动组合权重，却称模型参数稳健或生态有效；
- 情景名写“污染”，代码同时改变阻隔、死亡率和转化率；
- 论文方程、省略参数和脚本实现不等价；
- 确定性网格被写成“统计稳健性”；
- 完整 API 栏目先生成，再为全称、类路径和性能星级补写貌似合理的事实；
- 负稳健性结果只进入限制段，不回写摘要和结论。

### 6.4 应保留的工程写作特征

- 参数和输入输出表；
- 可复现命令、预算和停止条件；
- 校准、拟合外检查、敏感性和外部验证分层；
- 失败运行和负结果；
- 明确“当前参数组下模型产生”，不把模拟当现实事实。

## 7. 场景三：高端科研与期刊论文

### 7.1 来源家族

- BYD arXiv 方法论文、实验计划、论文版本和重写蓝图；
- Probet 前沿论文精读、方法迁移与研究决策笔记；
- 具备理论推导、创新主张、算法机制和期刊投稿目标的稿件。

课程教材中的高难证明仍归课堂场景，因为场景由文稿用途和读者合同决定，不由公式难度单独决定。其数学错误可以为科研规则提供反例，但不能把整份教材改判成期刊论文。

### 7.2 文体合同

目标是以最少但充分的结构承载创新主张。必须：

- 明确现有方法具体不能解释、测量或决定什么；
- 将创新拆成对象、机制、理论性质和经验结果，不用“新框架”统包；
- 对局部到全局、消元、包络、对象身份、定理条件和引用命题执行硬门禁；
- 允许论证有侧重，不强制每节等长、每段四格、每个结果都配对称优缺点；
- 保留反例、失败实验和无法唯一识别的结论；
- 让摘要、标题和贡献声明不超过正文最强证据。

### 7.3 人工观察到的专属 AI 痕迹

- `缺口-方法-贡献`在证据形成前就写成固定槽位；
- 每段强制 claim/evidence/boundary/transition，形成审稿预演腔；
- 用局部几何量直接解释全局优化性能；
- `存在常数、消元后、化简可得`隐藏承担结论的步骤；
- 文献同页邻近 OCR 被写成命题已经得到支持；
- 会议和蓝图中的 `主线、锁定、回滑、停止条件`残留到论文；
- 为显得完备而平均分配章节和论证篇幅；
- 模板化“首次、系统、统一、显著”创新词缺少最近工作对照。

### 7.4 应保留的科研写作特征

- 论证重点不均匀但证据完整；
- 关键构造和反例获得更多篇幅；
- 只在必要处声明边界；
- 术语稳定，不为避免重复而轮换概念名；
- 结果允许是条件性、局部性或否定性结论。

## 8. 禁止跨场景混用规则

| 错位操作 | 为什么错误 | 正确路由 |
|---|---|---|
| 要求课堂笔记证明“创新性”和最近工作差异 | 学习笔记没有期刊贡献合同 | 课堂 Skill 只要求教学目标、正确性和可复查性 |
| 删除课堂笔记全部步骤解释 | 把科研凝练误当教学质量 | 压缩例行步骤，保留关键选择和易错点 |
| 允许建模论文用“工程近似”跳过来源和误差 | 近似不是无证据许可 | 建模 Skill 要求近似理由、范围和敏感性 |
| 要求建模竞赛稿把每个经验关系写成形式定理 | 错用理论论文证明负担 | 检查模型-代码-数据一致性和决策有效性 |
| 允许科研论文沿用课堂口诀、题眼和“同理” | 教学捷径不能承担新知识主张 | 科研 Skill 展示决定性证明与条件 |
| 用建模参数敏感性替代科研机制验证 | 数值扰动不能证明理论机制 | 科研 Skill 分离理论性质、实验现象和机制解释 |
| 三场景统一套用“主张-证据-边界-建议”四格 | 不同段落功能被机械同构 | 各 Skill 按读者问题和证据负担组织 |

## 9. 共同底线与场景专属标准

以下底线不能因场景降低：

- 不捏造事实、数据、引用、作者经历；
- 不掩盖矛盾、负结果和条件不足；
- 数学对象、符号、定义域和基本等式正确；
- 继承题面、OCR、引文、官方答案和生成文字分段标记；
- 版本、权威文件和交叉引用可确定。

以下标准必须场景化：

- 解释密度；
- 段落长度和重复容忍度；
- 创新主张要求；
- 工程近似与验证语言；
- 公式推导展示粒度；
- 摘要、结论和学习回顾的结构；
- 对读者的提示语气。
