# 纯文风病灶目录

本文件只判断文本如何说，不判断所说内容是否正确。诊断时先观察跨段重复、篇幅分配和收尾习惯，再看单个词。单个“因此”不是问题；连续段落用相同路标和闭合动作，才构成模板病灶。

## 使用方法

每个病灶包含“识别信号、规则、案例和改写”。案例是结构范式，不要求逐字套用。改写时优先处理病灶形成的跨句关系，不做同义词替换竞赛。

## HUM-01 固定机械句首复用

**识别信号**

- 连续段落以“本文”“本节”“首先”“进一步”“需要指出的是”起句。
- 同级小节都以“问题 X 关注的是”“对于 X 而言”“从 X 角度看”开门。
- 多个分析段都以“依据/基于某理论”“根据某框架”起笔，只替换理论名称。
- 多段重复“面临的核心问题是”“核心挑战在于”“关键问题是如何”等问题报幕壳。
- 句首像字段名，换一节只替换名词槽位。

**规则**

`HUM-01 MUST`：三个相邻自然段不得靠同一语法槽位启动。根据段落的真实重心，分别从对象、矛盾、观察、选择、结果或限定起笔；若上下文已明确对象，可直接进入动作，不补路标。

理论或问题句壳单次出现不构成病灶。只有跨段重复且理论名称、问题标签只是可替换槽位时才处理；不得按固定比例限制理论段首，也不得把具体技术问题强改成反问。

**案例**

机器写法：

> 问题一关注资源恢复。问题二关注珍稀物种恢复。问题三关注食物链的长期变化。

拟人改写：

> 资源恢复先说。珍稀物种的恢复另起一段；食物链的长期变化放在最后。

**改写动作**

1. 圈出相邻五段的前十二个字。
2. 删除只负责报幕的句首。
3. 用每段最独特的信息占据首句。
4. 保留必要复现的术语，不机械轮换同义词。

**不要这样修**：把“本文”依次换成“本研究、该工作、此项分析”。词变了，槽位没变。

## HUM-02 万能过渡词堆叠

**识别信号**

- “因此、由此可见、值得注意的是、进一步而言、与此同时”密集出现。
- 过渡词删掉后，句意和前后关系没有变化。
- 每段结尾先总结一次，下一段开头再过渡一次。

**规则**

`HUM-02 MUST`：连接词只在关系无法由内容顺序自行显现时使用。因果、转折、递进和并列必须各自有具体语言根据；相邻两句已自然承接时，直接连接。

**案例**

机器写法：

> 值得注意的是，两个区间的曲线形态并不相同。进一步而言，这种差异说明参数的影响具有阶段性。因此，有必要分别讨论两个区间。

拟人改写：

> 两个区间的曲线形态不同，参数作用也随之分成两个阶段。下面分开讨论。

**改写动作**

- 先删过渡词，朗读前后两句。
- 若关系仍清楚，不补替代词。
- 若关系不清楚，用具体关系替代空路标，如“前一段比较末值，这里改看变化速度”。

**不要这样修**：把“因此”换成“故而”，把“值得注意的是”换成“不可忽视的是”。

## HUM-03 虚假逻辑转折、无必要衔接与强行串联

**识别信号**

- “虽然……但是……”两侧并不冲突。
- “不仅……而且……”只是把两个普通事实绑成对称句。
- 为了保持章节流动，每段都添加“这为下一节奠定基础”。
- 内容本可并列，却被强写成单线因果链。

**规则**

`HUM-03 MUST`：只保留能够改变读者预期的转折。并列内容直接并列，弱相关内容允许分段，章节之间不必伪造因果桥梁。

**案例**

机器写法：

> 虽然该方法结构简单，但是它可以用于后续计算。不仅如此，它还为下一部分的讨论奠定了坚实基础。

拟人改写：

> 该方法结构简单，可用于后续计算。

**改写动作**

1. 写出连接词声称的关系。
2. 若两侧没有真正冲突、递进或因果，删掉框架。
3. 用对象复现、变量回指或动作变化形成自然承接。

**不要这样修**：为了“更顺”给每段加一句承上启下。人类文本允许话题在标题处自然切换。

## HUM-04 固定“观点—支撑—边界—过渡”流水线

**识别信号**

- 每段都先宣告观点，再补两句说明，再写一句限制，最后引向下文。
- 不同重要性的内容拥有相同四步结构。
- 限定句像模板义务，不改变段落实际读法。

**规则**

`HUM-04 MUST`：段落结构由本段任务决定。定义段可以一句结束；关键判断段可以展开；结果段可以停在现象上；限制只在本段确有解释作用时出现；过渡只在读者需要方向提示时出现。

**案例**

机器写法：

> 本节分析参数变化。结果显示曲线先升后降。需要说明的是，该结果仅适用于当前设定。上述分析为下一节的机制讨论奠定基础。

拟人改写：

> 在当前设定下，曲线先升后降。下一节讨论这一转折。

或在无需过渡时：

> 曲线在中间区间达到峰值，参数继续增大后反而回落。

**改写动作**

- 为每段只写一个功能标签：定义、比较、解释、判断、例子、回指或停顿。
- 删除为了凑齐四格而出现的句子。
- 连续三段不得共享同一功能序列。

## HUM-05 段落长度高度均匀、句子长度过度平衡

**识别信号**

- 相邻五段的行数和句数近似相等。
- 每段固定三至四句，每句都在相似字数处停顿。
- 短信息被拉长，关键内容被压成同样尺寸。

**规则**

`HUM-05 MUST`：原文、文档职责或 Voice Profile 已显示信息地位不等时，让篇幅跟随该差异。允许一句段、两句段和长段共存；正式等权条件、定义组和标准条款保持平行。不得为了制造变化随机拆句或凭空指定重点。

**案例**

机器写法：三个并列小节各四句，依次介绍定义、作用、优势和小结。

拟人改写：原文未显示三个概念的主次，因此保持平行，只删除每节重复的“作用—优势—小结”槽位。只有原文或 Voice Profile 明确显示分量差异时，才调整篇幅。

**改写动作**

1. 按段标记 A/B/C 重要度。
2. 检查 A 段是否真的比 C 段获得更多解释空间。
3. 合并被拉长的低信息句，释放短段。
4. 把关键判断单独落句，但不要把所有结论都做成口号。

## HUM-06 全文节奏极度规整，缺少疏密与留白

**识别信号**

- 每节都以背景铺垫开始，以总结展望结束。
- 公式、解释、例子和提醒按固定周期循环。
- 读者无法从节奏感知哪里是重点。
- 同一段反复使用破折号承担解释、转折和强调，停顿不再跟随对象变化。

**规则**

`HUM-06 MUST`：在章节尺度设计“铺垫—加密—停顿”而非重复节拍。背景只在首次需要时出现；关键处允许连续推演；例行处允许压缩回指；一个具体判断后可以直接换节。

破折号、冒号和分号由语义关系与 Voice Profile 决定，不设机械配额。单个破折号、标题副标题和引语中的标点保持不动；只有同段集中复用并形成固定节拍时才分句或改回普通停顿。

**案例**

机器写法：每个公式后均有“其中……这表明……值得注意的是……”三句解释。

拟人改写：第一个公式完整交代符号；第二个公式只说明新增项；决定性公式前放慢解释，之后用一句指出变化，不再逐项复述。

**改写动作**

- 画出一页内的密度曲线：背景、公式、解释、判断、例子。
- 若曲线近乎平直，重分详略。
- 保留真正需要读者停下来的位置，压缩其余旁白。

## HUM-07 公文腔、汇报腔、审计腔、说教腔、辅导员腔和模板学术腔

**识别信号**

- 正文出现“抓手、闭环、落地、赋能、统筹推进、全面提升”。
- 用“必须指出、需要强调、读者应当注意”持续指挥读者。
- 用“完成、通过、放行、风险项、状态”汇报写作过程。
- 学术腔依靠“具有重要意义、提供有力支撑”而非具体表达。

**规则**

`HUM-07 MUST`：把管理动作改回研究对象、学习对象或工程动作。删除对读者的纪律性指令；把写作过程状态移出正文；评价词只有在句中承担具体区分时保留。

**案例**

机器写法：

> 本节围绕核心目标形成分析闭环，为后续方案落地提供有力支撑。需要强调的是，读者必须准确把握这一关键机制。

拟人改写：

> 本节分析该目标及其作用机制，结果供后续方案使用。

**改写动作**

- 圈出管理、验收、宣传和训导词。
- 问“句子实际说了哪个对象的什么变化”。
- 能回答就改成对象句；不能回答就删除。

## HUM-08 空泛学术套话和抽象名词堆砌

**识别信号**

- “系统性、科学性、有效性、协同性、复杂性”连用，却没有具体承载对象。
- “实现了对……的深入探讨”“具有一定的理论和实践意义”可复制到任何主题。
- “综合运用、系统梳理、提供理论支撑、有效解决、完善理论体系、不可或缺”成束出现，却不说明方法分工或结果。
- 多段重复“该处理体现了、该设计基于、该决策反映了”，处理名词自动推出抽象价值。
- 同段连续用“作为重要载体、扮演角色、发挥作用”绕开直接判断。
- 动词名词化，句子只剩“开展、实现、促进、提升”。

**规则**

`HUM-08 MUST`：将抽象评价还原成可读动作、差异或选择。若一个短语不能回答“具体多了什么、少了什么、换了什么、保留了什么”，删除或改写。

以上词项都只是上下文候选。单次“作为”、明确的变量角色、标准依据和具体方法分工可以 `KEEP`。改写“该设计体现了”时只能使用输入已有的原因和后果；不得像旧范例那样补造研究者曾经犹豫、失败或调整方案的过程。

**案例**

机器写法：

> 该框架把资源、捕食和外部扰动放进同一组状态方程，三类影响可以在一条时间轴上比较；这种组织方式兼顾了多维因素。

拟人改写：

> 该框架把资源、捕食和外部扰动放进同一组状态方程，三类影响可以在一条时间轴上比较。

**改写动作**

1. 标出以“性、化、度、机制、体系、框架”结尾的词。
2. 区分必要术语与空泛评价。
3. 用对象和动作改写，不强求每句都“学术”。

## HUM-09 创新性表演、空洞拔高和虚假升华

**识别信号**

- “首次、系统、统一、全新、突破、范式”密集承担气势。
- 普通组织选择被写成“创新框架”。
- 段末突然上升到治理、社会价值或方法论意义。
- 用“顺应时代—破局关键—持之以恒践行”把普通对象包装成口号式价值链。

**规则**

`HUM-09 MUST`：纯文风改写删除表演性强调，保留原文已有的具体差异表达。不要在本 Skill 中判断差异是否真实；只把“宏大评价先行”改成“具体动作先行”。

**案例**

机器写法：

> 本文首次将原先分开的三个子模型接到同一时间尺度上，并用共享状态量传递上游变化，由此构建系统、统一且具有突破性的分析范式。

拟人改写：

> 本文把原先分开的三个子模型接到同一时间尺度上，并用共享状态量传递上游变化。

**改写动作**

- 删除“首次/突破/范式”后检查句子是否还有内容。
- 把剩余内容前置；若没有剩余内容，整句删除。
- 不擅自补写“更准确的创新性判断”，因为那属于另一层任务。

## HUM-10 教科书式逐行公式解释与无效赘述

**识别信号**

- 公式后逐项把符号读成中文，重复符号表。
- 每次代入、移项、化简都配一条旁白。
- 图表中肉眼可见的升降被逐句复述。
- 关键选择和例行计算获得相同解释密度。

**规则**

`HUM-10 MUST`：解释非显然选择、变量角色变化和决定性转折；压缩重复运算、符号朗读和可见趋势。课堂场景可多保留一步，建模与科研场景应更快回到对象和结果。

**案例**

机器写法：

> 将式（3）代入式（4），可以得到式（5）。然后对式（5）进行整理，可以得到式（6）。进一步化简式（6），最终得到式（7）。

拟人改写：

> 将式（3）代入式（4）并整理，得到式（7）。

**改写动作**

- 标记每条解释是“选择说明”还是“操作播报”。
- 保留前者，合并后者。
- 若关键选择藏在三步操作中，把选择单独说清，其余计算回指公式。

## HUM-11 结尾强制总结、升华和展望

**识别信号**

- 每节都出现“综上所述，本节……”。
- 结论重复摘要、结果和讨论，没有新重心。
- 普通段落末尾强行写“具有重要意义”“为未来奠定基础”。
- 无实际后续内容也列“未来可进一步研究”。
- 多个案例段都以“此案例印证/揭示/挑战了”“从中可以看出”“这提示我们”点题。

**规则**

`HUM-11 MUST`：结尾只承担一种必要功能：落下判断、回收主线、指出尚未展开的问题或直接结束。没有新功能时删除结尾句。不得为了完整感自动生成总结、升华或展望。

案例点题尾句只有在重复复述前文时才处理。案例本身是正式研究对象、尾句引入新的具体差异或裁判要点时保留；不得把旧壳轮换成“问题的关键在于”“既然如此”等新的统一尾句。

**案例**

机器写法：

> 综上所述，本节通过上述分析全面揭示了该参数的重要作用，为后续研究奠定了坚实基础。

拟人改写：

> 删除该空壳句；若前文已有具体判断，让该判断直接收尾。

或直接删去整句，让图表后的具体判断成为结束。

**改写动作**

- 比较最后一句与本段前文；若只是复述，删除。
- 比较“展望”与正文；若没有具体未决动作，删除。
- 一个章节最多保留一个真正必要的回收段。

## HUM-12 无瑕疵、无侧重、无停顿的机器完美感

**识别信号**

- 所有问题都有完整答案，所有段落都有边界和收口。
- 作者从不跳过次要环节，也不暴露选择。
- 每个观点都被包装得同样圆满，文本没有局部粗细差。

**规则**

`HUM-12 MUST`：只通过原文或 Voice Profile 可确认的真实取舍打破完美感，而不是制造错误或作者立场。可省略重复解释并让具体判断自然停笔；“这里只看、其余沿用、不再展开”只是候选写法，重复使用时按 `HUM-16` 处理。

**案例**

机器写法：五个问题都用四段完成背景、方法、优点、限制和承接。

拟人改写：第一个问题因定义模型而详细；第二个问题只解释新增约束；第三、四个问题合并讨论共同参数；第五个问题直接围绕情景差异展开。结构差异来自问题职责，不来自随机打散。

**改写动作**

1. 写下作者真正最关心的一到两个位置。
2. 让这些位置获得解释峰值。
3. 合并重复的完整性部件。
4. 保留有限留白，不替读者完成所有归纳。

## HUM-13 句式对称过度与逻辑排布标准化

**识别信号**

- 高频使用“一方面……另一方面……”“既……又……”“不仅……还……”。
- 并列项句长、词性和评价完全镜像。
- “首先/其次/再次”“第一/第二/第三”或三个“从某维度看”机械复用，条目只为凑足三项。
- 优点、局限、建议总是等量出现。

**规则**

`HUM-13 MUST`：只有内容真正对举时才使用对称句。原文明确不等重时，重项可展开、轻项可并入从句；等权定义、条件、因素和标准条款使用 `KEEP`。并列数目由内容决定，不得为了 Humanize 人为破坏平行。

算法、证明、实验流程和规范步骤中的真实顺序不因命中枚举壳而改写。扫描器只给 `REVIEW` 候选；不得使用“每段最多一个三元结构”之类硬阈值。

**案例**

机器写法：

> 实验中最明显的变化是计算量下降；该方法同时提高了可解释性和适用性。

拟人改写：

> 计算量下降最明显；可解释性和适用性也有提高。

**改写动作**

- 拆掉成对连接词，比较两侧信息重量。
- 让最重要项占主句，次要项降为补充。
- 避免为了“三点完整”把一个意思拆成三条。

## HUM-14 统一化行文权重与每段均等用力

**识别信号**

- 背景、方法、结果、限制都使用相同篇幅和强调程度。
- 每个子问题都写同样多的“优点”和“局限”。
- 文本看起来全面，却很难复述作者究竟重视什么。
- 正文连续加粗多个“核心、关键、显著、重要”词组，用格式替代句位、篇幅和具体差异。

**规则**

`HUM-14 MUST`：从原文结构、用户目标或 Voice Profile 提取重心层级并让语言权重可见；不得由改写者自行指定 A/B/C。无法确认主次时使用 `REVIEW/NO_CHANGE`，不能让“反均匀”本身替代作者判断。

单次加粗、术语首次定义、表格字段和稳定列表标签可以保留。只有正文密集加粗且格式强调没有对应信息主次时才减少；不设全文五处或五十处等数量上限。

**案例**

机器写法：四个结果各写一段，结构都是“数值—解释—优点—限制”；原始结果中只有第二项出现反转。

拟人改写：先用一段比较四个结果，只把出现反转的第二项单独展开；其余三项在一句中交代共同趋势。读者一眼能看出真正改变判断的位置。

**改写动作**

1. 为章节列出一个主重心和不超过两个次重心。
2. 检查标题、篇幅、句长和例子是否体现该排序。
3. 合并等权重复段。
4. 删除“全面覆盖”造成的低价值解释。

## HUM-15 模态缓和堆叠

**识别信号**

- 一句内叠加“或许可以说、在某种程度上、一定的、可能、某些、相对而言”。
- 多个弱化词没有分别限定不同对象，只共同制造谨慎外观。
- 删除一个缓和词会改变强度，但无法判断应保留哪一个。

**规则**

`HUM-15 MUST`：区分真实认识强度与重复缓和。保留承担独立范围、概率或证据状态的模态；只有多个词限定同一命题且明显重复时才合并。无法确认强度时使用 `REVIEW/UNRESOLVED`，不得把“可能相关”改成“相关”。

**案例**

机器写法：

> 这一现象或许可能呈现初步规律，并可为后续分析提供某些线索。

拟人改写（仅在原意允许时）：

> 这一现象可能呈现初步规律，可作为后续分析的线索。

若“或许、可能、初步”分别承担明确强度，保持原文并标 `REVIEW`。

**改写动作**

1. 标出每个模态词限定的命题范围。
2. 合并只对同一命题重复降格的词。
3. 保留否定、条件、概率和未完成状态。
4. 用不变量检查器比较改前改后的模态标记。

## HUM-16 修复语模板化

**识别信号**

- 改写后多段以“这里真正、这里只看、只需、直接、其余沿用、不再展开”开头。
- 所有段落都被改成“对象先行 + 短判断 + 不总结”的同一新模板。
- 原作者惯用句法消失，场景默认声线取而代之。

**规则**

`HUM-16 MUST`：把修复语视为与原 AI 句壳相同的复用风险。相邻三段出现同一归一化修复骨架时重新改写或恢复自然原句。没有 Voice Profile 时采取最小改动，不用场景默认声线冒充作者个性。

**案例**

失败修复：

> 这里真正要看的是 A。这里只讨论 B。其余情形沿用前式，不再展开。

二次修复：

> A 决定本节的比较顺序。B 只在第二组出现；其他情形仍用前式。

**改写动作**

1. 对改后段首 4-8 字和句法槽位再次扫描。
2. 将修复短语纳入词库的重复窗口，而不是禁用一次出现。
3. 优先恢复作者原有自然承接方式。
4. 同版本复跑仍持续改写时标记非幂等。

## 组合病灶

真实文本通常不是单一病灶。优先按下列组合处理：

| 组合 | 表面症状 | 处理顺序 |
|---|---|---|
| `HUM-01 + HUM-02` | 起句和过渡都像批量生成 | 先删除报幕句首，再判断是否仍需连接词 |
| `HUM-04 + HUM-05 + HUM-14` | 每段四句、等长、等重 | 先重分段落职责，再分配篇幅，最后修句子 |
| `HUM-07 + HUM-08` | 管理词和抽象名词一起堆积 | 找回具体对象与动作，无法还原的信息直接删 |
| `HUM-09 + HUM-11` | 开头宣称创新，结尾强制升华 | 前后同时降载，让正文中的具体差异成为重心 |
| `HUM-10 + HUM-12` | 每一步都解释，文本无留白 | 确定解释峰值，例行步骤压缩回指 |
| `HUM-13 + HUM-14` | 并列完美、所有部分等权 | 打破假对称，主句只留最重要差异 |
| `HUM-15 + HUM-08` | 多重缓和包住空泛评价 | 先锁认识强度，再删除无信息评价 |
| `HUM-12 + HUM-16` | 用统一修复语制造“取舍感” | 回到 Voice Profile 与最小改动 |

## 诊断禁忌

- 不按关键词出现一次就判 AI 味。
- 不以句长方差、词频或段落长度单独判定作者身份。
- 不要求每段故意长短不同；变化必须有信息理由。
- 不把口语、俚语、第一人称或错别字当作“人类证据”。
- 不把内容层的问题偷换成文风病灶。
- “专家认为、研究表明、一些学者指出”等是归因来源言语行为。可以标记“当前句中来源不可见”供人工复核，但不得在纯文风层核验来源、删除归因后改成客观事实，或虚构作者、年份和引文。
