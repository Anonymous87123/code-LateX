# GENERAL 无收益改写失败复盘

## 盲评结果

fresh 代理在没有高风险词项命中的情况下改写 `cases/general/input.md`，统一验证器返回 `PASS/0`。随机化盲评中，改后稿为 A、原稿为 B；评审偏好 B：

| 文本 | 自然度 | 结论 |
|---|---:|---|
| 改后 A | 8.3 | 存在一处明显硬被动，不宜直接交付 |
| 原稿 B | 8.7 | 可直接交付 |

因此这是生成行为失败，不是 validator 的保护失败。`PASS/0` 只说明已编码门通过，不能覆盖成对读感。

## 逐处变化

有一定收益的变化很少：

- `均十分完整 -> 都很完整`：减少程式化程度；
- `直接滑到 -> 直接转向`：语域更中性，但也削弱原文对论证跳跃的判断色彩，不能算稳定收益。

无明确收益的形式化轮换：

- `从语言组织角度看 -> 从语言组织来看`；
- `这些都是好的学术表达习惯 -> 这些处理体现出良好的学术表达习惯`；
- `让模型选择……互相对应 -> 使模型选择……相互对应`。

明确退步：

- `造成论证前置过多 -> 使过多论证被前置`。

最后一处同时引入 `使 + 抽象对象 + 被动`、量词作用域歧义和翻译腔。更关键的是，原文真正略显生硬的“局限已经被解决”没有处理；代理把改动花在自然句上。

## 根因

v19 已有“改后须稳定优于原文”的段落级条款，但缺少三个执行抓手：

1. 改写前没有冻结具体病灶跨度，`BALANCED` 被执行成段内普遍清扫；
2. “更正式/更书面”可被模型当成收益，形式化同义轮换没有被排除；
3. 成对门只给整段最终判断，没有要求逐句回退无收益变化。

于是 validator 零 warning 被误当成质量证据，规则存在但没有阻止行为。

## v20 修复合同

`GENERAL` 现在执行最小改写面：

1. 改写前登记病灶原跨度、读感后果和允许动作；
2. 无可定位病灶即 `NO_CHANGE`，`BALANCED` 不是改动配额；
3. 每个改句须对应登记病灶并有独立收益；
4. “更正式、更书面、换个说法”、`让 -> 使`、自然主动因果改成 `使/让 + 抽象对象 + 被 + 动词` 不构成独立收益；
5. 不禁用合法技术被动句，只拦截相对原文新增且读感更差的硬被动；
6. 先逐句恢复无收益或劣化变化，再判断剩余整段；仍不能稳定胜过原文则 `NO_CHANGE`。

## fresh 验收

修复不能靠规则文本自证。使用完全相同的 `input.md/prompt.txt` 重新启动 clean-context 代理，并满足：

- 不读取旧输出、盲评或预期裁决；
- 若 `REWRITE`，随机化盲评必须偏好改后，且无新增硬被动；
- 若 `NO_CHANGE`，输入输出 SHA-256 必须一致，并说明没有足够稳定收益；
- validator 状态与实际 artifact SHA 对应；
- 不能用词项减少、压缩率或 `PASS/0` 代替盲评。
