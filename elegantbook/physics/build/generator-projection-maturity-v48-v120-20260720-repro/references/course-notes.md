## 1. 场景声音

课堂场景的自然感来自真实讲解取舍：教师或学习者知道哪里值得慢下来，也知道哪里一句就够。目标声音应当耐心但不包办，清楚但不把每个知识点加工成标准教学产品。

理想特征：

- 局部简化，关键处展开；
- 允许回指前文，不重复完整定义；
- 允许普通步骤连续写，不逐行配旁白；
- 允许一处轻提示，但不持续指挥读者；
- 允许短解、短段和自然停止；
- 作者判断通过“这里最容易混的是……”这类具体取舍表现；
- 不追求各节栏目对称。

课堂 Humanize 不是把文字改成聊天，也不是模拟老师现场说话。不要虚构“同学们已经发现”“老师课堂上强调过”或个人学习经历。

### 1.1 规则前置裁决

本文件的 `MUST/SHOULD` 只在连续阅读确认病灶后启动，不按单词自动执行。题干、直接引语、正式定义、定理条件、标准步骤、等权分类和用户锁定结构优先。真实证明链中的“因此/从而”和正式术语使用 `KEEP`；无法安全判断时使用 `REVIEW/NO_CHANGE`。

“一个解释峰值”是对不等权教学内容的建议，不得把本来等权的定义组或条件组强行改成主次结构。

### 1.2 语料动作锚

以下动作来自已登记、可读的课程 TeX 人工复读结果：`physics2.tex` 与 `physics1.tex`。两份来源当前均为 `origin_class=UNKNOWN`；人工复读不证明来源由人创作。因此四张卡必须标为 `PROVISIONAL`，COURSE profile 必须返回 `SUPPORTED_PROVISIONAL`。生产改写使用 `corpus_action_support=NONE`；只有独立候选审计才可选择这些卡，而选卡结果固定 `REVIEW/2`。它们只提供组织参考，不提供可复制句子；只有输入具备所列锚点时才有讨论价值。

只有未来接入代理不可伪造的外部 attestation，且经验证的独立人写来源达到门槛，COURSE 才可能变成 `SUPPORTED`。本地 `HUMAN_CONFIRMED` 声明仍为 provisional；即使未来达到门槛，也不自动证明 PERSONAL Voice、作者身份、学科正确或复制许可。

具体 provisional 卡和输入角色只保存在完整安装版的审计 catalog，不进入 generator projection，
也不在正常场景指南中复述。若公式、条件或纠偏本身存在内容矛盾，纯文风层保持输入命题并返回
内容门 `NOT_EVALUATED`；不能借未确认来源的组织动作偷偷修正学科结论。

## 2. AI 文风黑名单

### 2.1 句首和推进

高风险不是词本身，而是连续复用：

```text
于是 / 因此 / 接下来 / 现在看 / 再设 / 为此
这样一来 / 剩下只要 / 问题就落在 / 从而 / 综上
```

连续推导若每句都靠推进词开门，会像流水线工序。优先让函数、方程、图形、变量或当前动作直接出现。

### 2.2 固定教学槽位

警惕以下栏目在每节复制：

```text
章节定位 -> 核心规律 -> 解题方法 -> 典型例题
-> 常见陷阱 -> 一句话口诀 -> 本章小结
```

简单概念不需要完整七格。不同难度和用途的内容必须呈现不同结构。

### 2.3 多重记忆出口

同一内容同时出现“结论表、记忆总表、一句话口诀、最后背诵版、本章速记”，会让材料显得不肯结束。选择一个主要检索入口，其余位置回指。

### 2.4 教练、管理和模板学术腔

```text
阴招 / 秒杀 / 救命 / 掉坑 / 锁死 / 收割
整体地图 / 核心抓手 / 主入口 / 闭环 / 完整覆盖
重要内容 / 具有重要意义 / 系统掌握 / 全面提升
```

前一组制造考试表演，第二组把笔记写成项目管理，第三组是任何章节都能套的前言。改回具体对象和学习动作。

### 2.5 过度解释

- 同一概念依次用直观段、正式定义段和再次释义段完整说三遍；
- 每次代入、移项、换元和定义符号都解释目的；
- 公式已经给出关系，后文又逐字读一遍；
- 同一道题的两种解法都从题设完整重走。

### 2.6 强制闭合

- 末行公式之后再写“所以结论成立”；
- 每题都加“综上，故选……”；
- 完整讲义后再生成压缩版和背诵版；
- 每节自动补迁移、升华和下一节预告。

## 3. 课堂纯文风规则

### A. 结构与场景

`NOTE-HUM-01 MUST`：先区分理解概念、跟随推导、错题复盘、考前检索四种用途。不得把四种材料都写成完整教材章节。

`NOTE-HUM-02 MUST`：当定义、例子、计算、提醒和小结的信息地位明显不等时，只展开真正需要教学停顿的位置；本来等权的定义组和条件组保持平行。

`NOTE-HUM-03 MUST`：简单内容允许短解。若一句规则和一个例子足够，不补齐“定位—解析—陷阱—口诀”。

`NOTE-HUM-04 MUST`：难题允许局部变慢。关键构造、路线改变或容易混淆的位置展开，例行步骤连续写。

`NOTE-HUM-05 MUST`：相邻知识点不得复制相同栏目组合。根据本节独有任务保留二至四个必要部件。

`NOTE-HUM-06 MUST`：题目难度决定题解结构。简单题直接解；复杂题才给路线；只有存在真实迁移价值时才小结。

`NOTE-HUM-07 MUST`：同一知识只保留一个完整总结出口。正文、表格、口诀和速记中选择最适合检索的一处，其余只回指。

`NOTE-HUM-08 MUST`：第二种解法只从分叉点写起，只解释与主解不同的观察，不重抄题设和公共步骤。

`NOTE-HUM-09 SHOULD`：开头直接进入学习对象、识别动作或真正难点。删除“本章重要、考试常见、需要系统掌握”一类通用前言。

`NOTE-HUM-10 SHOULD`：标题只承担导航。若标题下面只有一两句且标题重复句意，合并到上一级段落。

### B. 句首、衔接和目的旁白

`NOTE-HUM-11 MUST`：连续三句若只是通过“于是/因此/接下来/现在/再”主持步骤，改由具体对象或公式关系承接；真实证明链和规范步骤使用 `KEEP/REVIEW`，不做连接词配额。

`NOTE-HUM-12 MUST`：过渡词必须承担真实的转折、因果、回指或步骤切换。删除后关系仍清楚时，不补替代词。

`NOTE-HUM-13 MUST`：禁止把“由此可见”换成“进一步而言”来制造变化。改关系，不换皮。

`NOTE-HUM-14 MUST`：一段最多保留一个必要的目的说明。只有路线选择不直观时，才用“为了/要得到/先看”。

`NOTE-HUM-15 MUST`：删除“现在问题落在”“剩下只要”“这样一来”等无信息主持语，直接出现待处理对象。

`NOTE-HUM-16 MUST`：“不是 A，而是 B”只用于真实且值得提醒的误读。普通定义直接陈述 B；但“不是死记结论，而是理解判断链”“不要只套公式，要先辨认条件”明确了学习动作或常见误读，必须保留其纠偏约束，只能压缩动员式语气。

`NOTE-HUM-17 SHOULD`：轻口语每处只承担定位、提醒或切换之一。无功能的“其实、说白了、别急、先来看”删除。

### C. 解释密度

`NOTE-HUM-18 MUST`：让公式承担已显示的信息。正文只解释非显然选择、对象角色变化或决定性转折。

`NOTE-HUM-19 MUST`：每次代入、换元或化简不自动生成旁白。把连续例行操作合成一句，把篇幅留给思路变化。

`NOTE-HUM-20 MUST`：直观、定义和识别方法可以并存，但不得三次完整复述同一内容。每层只提供新增信息。

`NOTE-HUM-21 MUST`：一处结论只封口一次。末行公式、答案句和总结句三者中保留最清楚的一处。

`NOTE-HUM-22 SHOULD`：已反复出现的套路允许回指，如“仍按上一例换元”。不要为了自足性把每题写成独立全文。

`NOTE-HUM-23 SHOULD`：口诀只有在明显压缩操作且便于检索时保留一条。不得自动押韵或给每节生成口号。

### D. 节奏、语气和作者感

`NOTE-HUM-24 MUST`：段落长短服从学习重点。允许一句提醒、短定义段和较长推导段并存。

`NOTE-HUM-25 MUST`：句长变化来自功能。定义和判断可短，关键解释可长；不得按固定字数随机拆句。

`NOTE-HUM-26 MUST`：允许合理省略重复解释和普通运算，不能通过故意语病、跳词或错别字制造粗糙感。

`NOTE-HUM-27 MUST`：持续命令语气改为具体观察、建议或直接说明。“必须/务必/需要/要 + 牢记/记住/记清楚”若只要求记忆而没有新增判断条件，属于同义教练壳，不能互相轮换。公式或规则已紧邻显示、命令句没有补充适用条件或误读时，删除整句；若命令承载独立操作要求，保留操作对象和强度。删除造成模态计数变化时保持 `REVIEW`，不得为获得 `PASS` 另造一个说教同义句。

`NOTE-HUM-28 MUST`：删除“阴招、秒杀、救命、锁死”等教练表演词。可以写“快速核对、常见误判、易混点”。

`NOTE-HUM-29 MUST`：“地图、抓手、闭环、入口、覆盖”等词只在承担课程管理腔时改写；若属于“知识地图、闭环控制、映射入口”等正式对象或用户术语，使用 `KEEP/REVIEW`。

`NOTE-HUM-30 MUST`：删除空泛学术前言和价值结尾。若一句不包含具体学习对象，不补同义套话。

`NOTE-HUM-31 MUST`：不伪造课堂互动或读者反应。把“聪明的同学已经发现”改成直接观察或一个真实问题。

`NOTE-HUM-32 SHOULD`：原文或 Voice Profile 已显示重点时，可用具体作者判断体现选择；“这里真正、这一段只需”等示例不得跨段复用，也不得凭空制造重点。

`NOTE-HUM-33 MUST`：不面面俱到。一个单元可以只解释一个难点，不自动补定义、正反例、陷阱、口诀和练习。

`NOTE-HUM-34 MUST`：完成学习动作即可停下。章末小结只回收最可迁移的一点，不复述目录，也不强制展望。

`NOTE-HUM-35 SHOULD`：连续三组及以上“短中文字幕单独占行，下一物理行只放独立 TeX 公式”只作为结构复核候选，检查这些字幕是否把连贯推导切成机械卡片。一两步正常推导、正式定义/命题/定理和受保护环境不命中；候选本身不得触发 `FAIL`，也不授权删除字幕或公式。单个“因此/于是/从而”不是 AI 病灶；对象标签、条件、因果、推论和递进连接承担语义功能时必须保留。若确需取消独占字幕，先把同一逻辑关系并入相邻正文或公式引导句；只让前后公式相邻不算保留关系。scanner candidate 不得直接映射为 `DELETE_STYLE_SHELL`。

## 4. 节奏调控

### 4.1 概念讲解

推荐节奏不是固定模板，而是：先给对象，再在一个容易混淆处放慢，最后给一个可操作识别法。若概念非常简单，定义和识别法可以合成两句。

### 4.2 长推导

将推导分成三种速度：

| 位置 | 速度 | 写法 |
|---|---|---|
| 路线选择或关键构造 | 慢 | 解释为什么此处换视角或引入对象 |
| 例行代数 | 快 | 连续公式或一句“代入整理” |
| 结论转折 | 停 | 用短句指出新的对象关系，然后继续 |

不要让整段都慢，也不要把决定性选择藏在匀速旁白里。

### 4.3 例题

- 主解完整到足以理解路线；
- 次解只写差异；
- 易错点放在第一次可能误读的位置，不另建“陷阱大全”；
- 答案落下后可以结束；
- 只有当例题真正提供下一题可复用的识别动作时，才写一两句小结。

### 4.4 考前卡片

考前卡片可以密集、列表化和直接，但不要同时保留完整教材叙述。卡片只负责检索，不伪装成系统讲解。

## 5. 机器完美感破除

合理破除方式：

- 某节只有一个例子，没有自动反例；
- 某题直接结束于公式或答案；
- 两种解法一详一略；
- 章节之间段数不同；
- 同类公式首次完整解释，后续只回指；
- 次要知识用一句带过；
- 一个真正难点占据全页大部分解释。

禁止方式：

- 故意漏字、写错别字或制造病句；
- 随机删掉必要的承接词；
- 批量加入“其实、说白了、简单来说”；
- 虚构老师、同学和个人学习经历；
- 为了长短变化把完整句硬切成碎片。

## 6. 子场景差异

| 子场景 | 重心 | 可省略 | 最危险 AI 味 |
|---|---|---|---|
| 随堂笔记 | 当堂概念与一个关键观察 | 完整背景、系统总结 | 每点都加工成教材条目 |
| 教材讲解 | 概念层次与代表例题 | 重复定义、同型第二例 | 七格教学模块循环 |
| 题解 | 路线变化与决定性步骤 | 例行计算、第二解公共部分 | 每步旁白、三重封口 |
| 错题复盘 | 误判发生的位置与改法 | 重新抄整道标准答案 | 把复盘扩成完整知识体系 |
| 考前卡片 | 快速检索和少量提醒 | 长背景、完整推导 | 口诀、救命表和刺激性教练腔 |

## 7. 终审问题

1. 简单内容是否真的短，难点是否真的慢？
2. 相邻知识点是否仍使用同一栏目和同一开头？
3. 同一知识是否仍有多个完整总结出口？
4. 第二解法是否只写差异？
5. 公式旁白是否提供新增信息？
6. “于是/因此/接下来”是否还在批量轮换？
7. 教练腔、管理腔和通用章节前言是否已经退出？
8. 是否存在自然短段、回指和不总结的结尾？
9. 能否从篇幅看出作者认为什么最值得学？
10. 所有检查是否都停留在文风，没有判断内容对错？
