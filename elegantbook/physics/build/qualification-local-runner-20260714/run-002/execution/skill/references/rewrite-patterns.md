# 分场景去模板化改写范式

本库从旧场景 Skill 的 96 个混合模式中剥离纯文风动作，并结合真实 MD/TEX 的人工阅读重新组织。这里不继承任何内容审核门。每个模式只处理句式、节奏、结构、腔调或机器完美感。

模式中的“改前/改后”是可迁移范例，不是固定模板。使用时保留输入语义，不逐字套句。

范例优先级低于保护区、语义不变量、Voice Profile 和强度权限。以下情况默认不改或先 `REVIEW`：正式定义/命名、结果报告、直接引语、等权条件、标准条款、真实证明链和正式术语。不得把“本文称为 X”按对象先行改成“X 是……”，也不得为了篇幅坡度凭空指定中心项。

改写后必须检查范例语言是否成为新句壳，尤其是“这里真正、这里只看、只需、其余沿用、不再展开”。

## 1. COURSE：课堂笔记与学习复盘

### CP-01 通用章节前言 -> 具体学习动作

**改前**：解析几何是高中数学的重要内容，在考试中占有重要地位。本章从直线开始，后面的圆和圆锥曲线都先把图形条件写成方程。

**动作**：删除价值宣告，让章节独有对象和动作先出现。

**改后**：解析几何先做一件事：把图形条件写成方程。这里从直线开始，后面的圆和圆锥曲线都要用到这一步。

### CP-02 “不是背诵，而是训练” -> 直接给操作

**改前**：本讲义的核心不是背零散结论，而是训练你看到代码后判断对象何时出生、何时拷贝、何时死亡。

**动作**：删掉假想错误读法，直接进入复习动作。

**改后**：看这类代码，先追对象的生命周期：在哪里创建，是否发生拷贝，离开哪个作用域时销毁。

### CP-03 整体地图话术 -> 本章独有难点

**改前**：先建立整体地图。本章表面上知识点很多，其实可以压成两条主线：对象怎样创建和销毁；基类指针最终调用谁的函数。

**动作**：用本章真正容易混淆的对象替代“地图、主线”。

**改后**：这一章最容易混的是两条线：对象怎样创建和销毁；基类指针最终调用谁的函数。

### CP-04 固定六格讲义 -> 各节独立重点

**改前**：每节依次写结构规律、解题方法、答案信号、例子、干扰项和总结。

**动作**：取消统一栏目，按本节独有价值组织。

**改后**：取消固定六格；长对话、短文和讲座分别只保留与当前任务有关的栏目，不预设具体重点。

### CP-05 多重记忆出口 -> 一个权威汇总

**改前**：正文结论之后依次给记忆总表、一句话口诀、最后背诵版和本章速记。

**动作**：保留最适合检索的一处完整汇总，其他位置只回指。

**改后**：保留“记忆总表”作为唯一汇总。正文讲到具体运算符时写“见表 1”，章末不再生成第二套口诀。

### CP-06 匀速代数旁白 -> 关键转折处放慢

**改前**：把 (x,y,z) 代回原式，得到……再用 (x+y=1/2) 化简……于是……结合范围可保留下来……

**动作**：合并例行代入，只在数学视角发生变化时解释。

**改后**：合并代入与化简旁白，原有数学式保持不变；只在原文已经转而把 (x,y) 视为二次方程两根的位置保留解释。

### CP-07 三层推进词 -> 对象直接承接

**改前**：这样一来，(x,y) 就是两个实根。要让实根存在，判别式必须非负，于是……

**动作**：删除主持语，让对象关系承担因果。

**改后**：(x,y) 是该二次方程的两个实根，因此
\[
\Delta\ge 0.
\]

### CP-08 目的旁白串 -> 一次说清路线

**改前**：乘积难以比较，于是把三根看成公共值的函数。若能证明……那么……因此接下来需要……

**动作**：保留一次路线选择，删掉连续的“若能—那么—因此”。

**改后**：把三根写成公共值 (c) 的函数，并记 (P(c)=x_1(c)x_2(c)x_3(c))。只需观察 (P) 随 (c) 的变化；端点随后给出比较。

### CP-09 符号定义字幕 -> 合并成数学句

**改前**：由上式可知 (q(x)<0)。因此令 (s(x)=-q(x)>0)，于是再令 \(\phi(x)=1/s(x)\)。

**动作**：符号定义直接并列，不为每次定义配完整旁白。

**改后**：因 \(q(x)<0\)，记 \(s(x)=-q(x)>0\) 及 \(\phi(x)=1/s(x)\)。

### CP-10 直观、定义、释义三遍 -> 每层只增量

**改前**：先用一段解释“交换变量不变”，再给正式定义，定义后又说“变量作用完全相同”。

**动作**：保留例子、正式定义和一句识别法，删除第三次同义解释。

**改后**：例子后给定义，再补一句：“判断时交换任意两个变量，式子不变即可。”

### CP-11 第二解法完整复制 -> 从分叉点写差异

**改前**：坐标解法完成后，以“另一种解法”重新抄题、设点并走完整推导。

**动作**：主解完整，次解从新观察出现处起笔。

**改后**：另一种解法从与主解分叉的首个观察写起；题设和公共步骤回指主解，不补造未提供的圆、曲线或计算路线。

### CP-12 教练表演腔 -> 低刺激检索语言

**改前**：秒杀题型、救命表、考试阴招、千万别掉坑。

**动作**：保留快速检索功能，去掉情绪动员。

**改后**：快速核对、填空对照表、常见误判。不添加原文未提供的解题动作。

### CP-13 三重封口 -> 一次结束

**改前**：末行公式给出答案，随后写“所以结论成立”，再写“综上，故选 ACD”。

**动作**：保留一个最清楚的结束位置。

**改后**：故选 A、C、D。

### CP-14 完整版 + 压缩版叠加 -> 按使用时机二选一

**改前**：完整讲义后附“最后背诵版”，随后再压成一句。

**动作**：理解材料保留讲解和一张表；考前卡片直接使用压缩版。

**改后**：正文用于理解时，章末只留一张对照表。材料用于考前检索时，直接提供卡片，不叠加完整教材结构。

### CP-15 简单知识也六格 -> 短内容短写

**改前**：每个知识点都有考法定位、典型题、答案、解析、坑点和速记。

**动作**：让结构随复杂度变化。

**改后**：简单知识只保留原文已有的一句规则和一个例子；只有原题确实需要多步追踪时才使用完整解析。

### CP-16 作者判断缺失 -> 标出解释峰值

**改前**：依次介绍条件、代入、化简、判别式和选项判断，篇幅相同。

**动作**：明确本题真正需要停留的位置，压缩其余步骤。

**改后**：只在原文已明确显示的转折处展开，其余代入和化简快写；不凭范例指定 (x,y) 或判别式为新重心。

### CP-17 连续命令读者 -> 观察与建议

**改前**：必须记牢这张表。做题时先把这一步固定下来，千万不要直接代入。

**动作**：删除纪律性语气，保留实际操作建议。

**改后**：这张表用于区分原文已列出的情形。做题时先执行原文给出的第一项检查，不额外补造对象创建或拷贝步骤。

### CP-18 口诀自动生成 -> 只保留有效压缩

**改前**：每节末尾都写一句押韵口诀，即使口诀只是把定义换序重复。

**动作**：检查口诀是否缩短检索路径；没有则删除。

**改后**：只有符号顺序容易混淆的章节保留一条对照句，其余章节直接结束于例题。

### CP-19 每节都有迁移小结 -> 只在有新识别法时回收

**改前**：本节方法可以迁移到更多问题，具有广泛适用性，读者应熟练掌握。

**动作**：删抽象迁移评价；若存在具体识别动作，只写该动作。

**改后**：若正文已给出具体可迁移的识别动作，结尾只保留该动作；否则删除整句，不从范例引入新方法。

### CP-20 全文匀速 -> 建立“慢—快—停”

**改前**：关键构造、例行代数、结果判断均用两句解释。

**动作**：关键构造长写，代数快过，结果短句落下。

**改后**：构造处解释选择；中间三行公式连续呈现；最后用一句“范围由判别式决定”结束，不再加总结段。

## 2. MODELING：数学建模与工程论文

### MP-01 “表格的作用” -> 表格实际内容

**改前**：表格的作用在于让读者一眼看出每个参数来自哪条事实链。

**动作**：删除对媒介职责的说明，直接说表格列了什么。

**改后**：表 2 列出各参数及其事实来源。

### MP-02 “正文里要保留” -> 正文已经呈现的对象

**改前**：正文里既要保留粗扫结果，也要保留加密扫描结果。

**动作**：把编辑指令改成已完成的正文陈述。

**改后**：正文同时给出粗扫结果和加密扫描结果。

### MP-03 “扫描的写法” -> 扫描动作

**改前**：参数扫描的写法也要服务于结论表达。粗扫负责定位区间，加密扫负责确认……

**动作**：删写作评价，保留计算顺序。

**改后**：粗网格先定位变化区间，随后用局部加密网格缩小范围。

### MP-04 “如果放到图上” -> 图中已有标记

**改前**：如果把它放到图上展示，还应在曲线旁标明扫描范围和加密区间。

**动作**：从假设性编辑语气改成正式图表陈述。

**改后**：图中标出扫描范围和加密区间。

### MP-05 “优点不在于” -> 直接说明模型取舍

**改前**：本问的优点不在于继续增加模型维度，而在于把设定、校准和比较统一成闭环。

**动作**：删除自我评价和闭环词，保留具体组织动作。

**改后**：本问统一组织设定、校准和比较，不增加原文未提供的模型类型或比较对象。

### MP-06 “局限 + 后续补强”固定尾巴 -> 一次直接说明

**改前**：其局限同样需要写清……后续若继续补强，应优先……

**动作**：把真实限制压成一句；后续动作只有必要时集中出现。

**改后**：把原文已明确给出的局限直接压成一句；原文没有具体后续对象时，删除后续句。

### MP-07 “五问不是孤立” -> 写清共享对象

**改前**：五个问题不是孤立求值，而是沿着底层资源—中层鱼群—珍稀物种—复杂食物链—综合情景展开。

**动作**：删除宏观链条表演，说明哪些对象真正传递。

**改后**：五个问题依次处理底层资源、中层鱼群、珍稀物种、复杂食物链和综合情景。不从这个列表推断额外的状态传递或动力学关系。

### MP-08 双重“不是，而是” -> 直接陈述方法问题

**改前**：本文关注的不是传统分组问题，而是构造一种分组，使同组变量的局部曲率尺度尽量接近，从而降低诱导子问题的尺度失衡。

**动作**：删除两层纠偏，保留研究动作。

**改后**：本文构造一种分组，使同组变量的局部曲率尺度尽量接近，从而降低诱导子问题的尺度失衡。

### MP-09 “也就是说”复述公式 -> 一次概括

**改前**：控制组内对数曲率跨度。也就是说，这本质上就是在控制组内条件数上界。

**动作**：把关系放在一个句子中，不先说一遍再翻译一遍。

**改后**：在对角二次型上，组内对数曲率跨度对应条件数的对数上界。

### MP-10 “之所以……最自然稳定” -> 设计造成的直接变化

**改前**：之所以采用排序后连续切块，是因为这是最自然也最稳定的做法。

**动作**：删主观优越评价，直接写该设计避免了什么。

**改后**：本文采用按 (s_i) 排序后连续切块的方法。原文未提供稳定性或避免何种失衡的证据时，不补写该效果。

### MP-11 “三个好处”凑整 -> 分散到首次作用处

**改前**：这样定义有三个好处：一……二……三……

**动作**：若三项不等重，不集中凑列表；分别在变量首次发挥作用时说明。

**改后**：三项作用只在各自首次发生时说明；不从范例引入 `overflow`、安全段或目标函数。

### MP-12 项目经理式“先验证掉” -> 工程动作

**改前**：第一步不是全量大跑，而是最小验证。先把接口和输出验证掉。

**动作**：保留务实顺序，删除内部命令声线。

**改后**：先验证接口和输出，再运行全量计算。

### MP-13 “能不能开始做？能” -> 定稿陈述

**改前**：如果只问一句“这版能不能开始做了”，答案是：能。

**动作**：删除聊天式自问自答，陈述当前文本具备的内容。

**改后**：该版本可以开始实施。若需说明依据，只列出原文已给出的实现要素。

### MP-14 摘要逐问报到 -> 共享主干 + 重点输出

**改前**：首先建立问题一模型；其次分析问题二；再次处理问题三；最后讨论问题四和问题五。

**动作**：找出贯穿全文的模型主干，只保留最有区分度的结果。

**改后**：摘要围绕各问共享的模型主干展开，只保留原文已有的关键取舍和结果；其余子问题移入正文。

### MP-15 五问复制“优点—局限—补强” -> 各问异构

**改前**：每问末尾都有“本问优点、其局限、后续补强”。

**动作**：问题一写建模取舍，问题二直接比较响应，问题三解释分类，问题四报告变化，问题五只保留排序；共同限制集中全文末段。

**改后**：五问段落结构不再对应，篇幅由各问新增的信息决定。

### MP-16 抽象框架名词串 -> 具体对象与动作

**改前**：该框架通过模块化机制，把资源、鱼群和外部扰动写进同一组状态方程，形成完整链条和统一口径。

**动作**：拆开抽象名词，指出哪些对象被放到一起。

**改后**：模型把资源、鱼群和外部扰动写进同一组状态方程，各情景只改变相应项。

### MP-17 每个公式都解释“作用” -> 只说新增项

**改前**：给出资源方程后逐项读符号；第二个方程再次解释全部变量，其中沿用前述增长项，只新增捕捞压力 (H)；随后说明两式共同构成闭环。

**动作**：首式交代共享符号，后式只解释新增量和当前用途。

**改后**：第二式沿用前述增长项，只新增捕捞压力 (H)。后文的对照情景通过改变 (H) 构造。

### MP-18 每图都“结果表明” -> 观察直接开场

**改前**：结果表明，图 6 中情景 A 的曲线先上升后下降。值得注意的是……

**动作**：把曲线真正的转折放在句首，图号后置。

**改后**：情景 A 的曲线先上升后下降（图 6）。

### MP-19 每节自动未来工作 -> 集中一次

**改前**：每个子问题末尾都写“后续可进一步细化、补充更多数据或扩大扫描”。

**动作**：删除自动未来句，只在结尾保留一个真正优先的后续动作。

**改后**：各小节在结果处结束；全文结尾只保留原文已明确给出的一个后续动作，没有具体对象时不补写。

### MP-20 结论逐问验收 -> 一个中心判断

**改前**：问题一说明……问题二说明……问题三得到……问题四验证……问题五表明……综上……

**动作**：不回放目录，选择最能改变读者理解的判断。

**改后**：结论不再逐问复述，只保留原文已显示为中心的判断；原文未显示唯一重心时，保持现有并列，不凭范例强选。

## 3. RESEARCH：高端科研与期刊论文

### RP-01 连续否定纠偏 -> 一次对比后正面陈述

**改前**：“超越交互”并非否定变量交互的重要性，而是指出……本文并不试图以局部条件性取代交互，而是将其视为……

**动作**：保留一次真正对比，删除作者持续辩护的外壳。

**改后**：交互恢复仍是变量分组的基础，但它不足以描述分组后子问题的数值形态。本文把局部条件性作为另一条评价维度。

### RP-02 方法节先汇报写法 -> 研究对象推进

**改前**：在此基础上，证据组织不采用并列铺开，而是按一条逐层收紧的论证路径展开……

**动作**：删除文章组织旁白，陈述对象如何变化。

**改后**：删除证据组织方式的旁白，直接写原文已有的第一个研究对象及其变化；不从范例引入求解器、分组或预算设定。

### RP-03 反对意见预演 -> 研究问题层次

**改前**：即便我们确认了这一点……即使存在这种关联，也仍可能……如果要排除……同时预算也可能……

**动作**：保留问题层次，删除逐项审稿答辩。

**改后**：把原文已给出的问题层次集中写成一个正面范围句；输入未提供具体研究问题时，不补写分组、终点或预算层次。

### RP-04 结果节五层路线旁白 -> 最强观察开场

**改前**：本节依次呈现五层证据，各环节均由前一结论揭示的缺口驱动……

**动作**：取消目录式说明，直接进入最有辨识度的结果。

**改后**：删除“五层证据”路线旁白，直接以原文已报告的最强观察开场；不从范例引入 F1、Hessian 或分组结论。

### RP-05 “这一层的作用不是” -> 当前问题
