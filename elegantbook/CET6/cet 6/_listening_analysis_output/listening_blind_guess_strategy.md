# 六级听力选项盲推策略：只看选项时怎么提高胜率

## 研究口径

这份文件专门研究“完全不看听力原文，只看 A/B/C/D 选项，能不能盲推”的策略。

数据口径：

- 本文只保留 2022-2025 共 425 题的最终选项与答案。
- 425 题全部纳入规则统计、信号统计和相近项统计。

## 总结论

这份文档能支持的不是“固定口诀”，而是四层更稳的判断：

1. **先排包装，再看关系**。长且字母多、数字壳、研究壳、泛化抽象壳、专名壳，通常先降权。
2. **短项只是弱加分**。短要和“关系完整、能接住题问”一起看，不能因为短就直接选。
3. **Section 真正决定看法**。A 看对话推进，B 看段落功能，C 看概念链和相近候选簇。
4. **相反项看维度，相近项看候选簇**。相反项不是答案保证；相近项在 Section C 里更像“答案可能就在这两项中”的提示。

| Section | 更像答案 | 更像干扰 |
| --- | --- | --- |
| A | 短而完整、能接前后题、末题 future/plan 但要先有对话推进 | 最长且字母最多、身份标签、立场偷换、把另一位说话人的立场套过来 |
| B | 段落功能、结果/建议、解释链 | 数字壳、研究壳、局部例子、过强绝对；相近项如果只是同类包装就不稳定 |
| C | 发现/机制/影响链、能概括结论的评价、相近对里的短而准选项 | 专名、学术壳、只有抽象词没关系、只说研究方法没有发现 |

最重要的发现：

1. **最长项明显不可靠**：词数唯一最长命中 `48/247`，约 `19.4%`；字符唯一最长命中 `40/351`，约 `11.4%`；字符唯一最长且字母唯一最多命中 `31/333`，约 `9.3%`。
2. **最短项有弱优势，但不是固定公式**：词数唯一最短命中 `67/220`，约 `30.5%`；字符唯一最短命中 `151/359`，约 `42.1%`；字母唯一最少命中 `157/360`，约 `43.6%`。
3. **字母数量比肉眼长短更细一点**：字母唯一最多只命中 `34/358`，约 `9.5%`；“长且字母多”更适合排除，不适合硬押。
4. **数字壳、研究壳、泛化抽象壳大多是包装**：唯一数字项 `10/48`，研究项 `1/17`，泛化抽象项 `8/39`，都不适合当单独直选信号。
5. **因果词和未来词不能机械化**：唯一因果词项 `2/16`，唯一 future 项 `3/16`；A 末题 plan/future 规则也只有 `1/6`，所以必须回到 section 语篇链里看。
6. **真正有用的是“完整关系”**：抽象核心词整体 `18/62`，Section C 中 `9/29`；它只能说明“有可能是概念链”，不能替代题问对象。
7. **表述相近项值得保留为候选簇**：相近对阈值 `>=0.24` 时，答案在这对里 `73/108`，约 `67.6%`；Section C 是 `39/53`，约 `73.6%`。

一句话：

> 盲推不是找最像答案的华丽选项，而是先把包装壳剔掉，再在对应 section 里挑那个最短、最完整、最能接住前后题的选项。

## 盲推优先级

如果完全没听到，建议按下面顺序处理：

1. 先排除“字符最长 + 字母最多 + 信息堆砌”的选项。
2. 再降权唯一数字、唯一研究壳、唯一具名例子、泛化抽象壳；因果词要先看是不是在回答原因/结果。
3. 若某项字母数/字符数都明显更短，而且关系完整，优先保留。
4. 若出现一对表述明显相近的选项，先把它们当候选簇；Section C 中尤其要重点比较这对。
5. 若题号是 Section A 的身份/状态/计划题，简短直接项更可疑。
6. 若选项里有唯一未来/计划项，尤其在对话末题，只能在它接得住对话推进时提高优先级。
7. 如果前两题已经同字母，第三题完全没信息时，降低继续同字母的优先级。
8. 如果没有明显结构差异，再用题号位置和 section 先验做最后选择。

## 规则 1：不要迷信最长项

425 题中：

- 425 题里，单看词数最长：`247` 次出现，命中 `48` 次，约 `19.4%`。
- 425 题里，选“字符唯一最长”：`40/351`，约 `11.4%`。
- 选“字母唯一最多”：`34/358`，约 `9.5%`。
- 如果某项同时是“字符唯一最长 + 字母唯一最多”：`31/333`，约 `9.3%`。

说明：

最长项常常不是“更完整”，而是“把干扰信息写得更像回事”。六级听力很多正确项反而是对原文题点句的简洁概括。

典型情况：

- A. `He is a staff writer`
- D. `He is a father of four kids`

D 更长，但答案是 A。

盲推策略：

- 如果一个选项明显最长，而且包含很多附加名词、限定、背景，它更像干扰项。
- 只有当最长项表达的是完整逻辑关系，而其他项只是零散名词时，才保留它。

## 规则 2：唯一最短项有弱优势，但不是铁律

425 题中：

- 425 题里，单看词数最短：`220` 次出现，命中 `67` 次，约 `30.5%`。
- 425 题里，选“字符唯一最短”：`151/359`，约 `42.1%`。
- 选“字母唯一最少”：`157/360`，约 `43.6%`。
- 选“单词唯一最少”：`67/220`，约 `30.5%`。
- 按 section 看，字符唯一最短在 A/B/C 中约为 `44.3% / 36.3% / 44.4%`；字母唯一最少在 A/B/C 中约为 `45.5% / 36.7% / 46.7%`，所以它不是 Section A 独有规律。

为什么最短项会略占优：

1. 对话题常考身份、状态、态度，答案可能是简洁概括。
2. 干扰项常需要加修饰词才能“伪装合理”。
3. 正确项常是原文关系的压缩表达。

但要注意：

- 最短项如果是显眼否定项、具名例子项、过强绝对项，仍可能是坑。
- Section C 里短项也可能命中，但必须是完整概念关系，不能只是一个孤立名词。

盲推策略：

> 没听到时，Section A 中“唯一最短且不是过强/具名例子/奇怪否定”的选项，可以作为优先候选。

## 规则 3：字母数量适合做“强排除”

网上常说“Section A 和选项字母数量有关”。把选项里的题组提示、页眉、答题卡说明先清掉后，再分别看字符数、英文字母数、单词数，结论更像这样：

- 选“字符唯一最短”：`151/359`，约 `42.1%`。
- 选“字母唯一最少”：`157/360`，约 `43.6%`。
- 选“单词唯一最少”：`67/220`，约 `30.5%`。
- 选“字母唯一最多”：`34/358`，约 `9.5%`。
- 如果某项同时是“字符唯一最长 + 字母唯一最多”，只有 `31/333` 是答案，误伤率约 `9.3%`。

这说明两件事：

1. **短项可以加分，但不能机械选**。短项优势大概是 42% 到 46% 左右，不是 70% 或 80%。
2. **长且字母多更适合排除**。它常常是把背景、限定、对象、结果全塞进去的包装项。

考场用法：

- 先数大概，不需要精确数到每个字母：哪一项明显长一截，先打问号。
- 如果某项又长、又有数字/研究/因果/专名，更像诱饵。
- 如果某项短，但只是一个孤立名词、身份标签或否定陷阱，不因为短就选。
- 如果某项短，而且已经把“谁做什么/什么导致什么/态度是什么”说完整，才给它加分。

## 规则 4：唯一因果词项不要轻易选

425 题中：

- 唯一因果词项：`16` 次出现，命中 `2` 次，约 `12.5%`。
- 它更像“关系提醒”，不是答案保证；更准确的说法是：它必须回到题目是不是在问 `why/result` 来判断。

因果词包括：

- `because`
- `reason`
- `cause`
- `lead to`
- `result`
- `stem from`
- `why`

为什么容易是干扰：

因果词很显眼，命题人常把“解释性语言”写成一个看似高级的选项，但题目实际问的可能是事实、建议、影响或结论，不是原因。

盲推策略：

- 如果四个选项只有一个显式写出 `reason/cause/result`，先看它是不是把“解释壳”塞错了位置。
- 如果题目本来就在问原因、机制、结果链，因果词反而是定位信号，不是自动排除信号。

## 规则 5：唯一数字项更像诱饵

425 题中：

- 唯一数字项：`48` 次出现，命中 `10` 次，约 `20.8%`。
- 按 section 看，A/B/C 分别是 `3/13`、`1/10`、`6/25`；作为排除项仍然有用，但更适合当“先降权”的提醒，而不是看到数字就排。

数字很容易被听到，但盲推时数字项常有两个问题：

1. 数字是背景信息，不是题目问点。
2. 数字对应的是相邻题或同段其他信息。

盲推策略：

- 如果只有一个选项含数字，不要因为它具体就选。
- 如果四个选项都含数字/比较，数字才成为题眼。

## 规则 6：唯一研究词项不值得盲选

425 题中：

- 唯一研究词项：`17` 次出现，命中 `1` 次，约 `5.9%`。
- 这条更像“壳”，不是自动答案；如果它没有说出研究发现本身，优先降权。

研究词包括：

- `research`
- `study`
- `experiment`
- `scientist`
- `survey`
- `data`
- `methodology`
- `journal`

为什么不可靠：

Section B/C 本来就大量出现研究背景。选项里有 `study`，不等于它是答案。真正答案通常是研究发现的内容，而不是“研究”这个壳。

盲推策略：

> 看到唯一研究词项，先问：它有没有说出 found/showed/suggested 的具体内容？如果只是“研究/方法/数据”本身，更像壳。

## 规则 7：唯一具名例子项先降权，不写成铁律

具名例子项出现次数很少，不能写成稳定规律：

- 唯一具名例子项：`16` 次出现，命中 `4` 次，约 `25.0%`。
- 不能把它写成铁律；更合理的用法是：如果它只是具体人物/地点/实例，而没有把关系说完整，就先降权。

具名例子包括：

- Boston
- rats/mice
- hospital/doctor/nurse
- millionaires
- apps
- bears

盲推逻辑：

主旨题、结论题、讲座题常用例子支撑观点。具体例子容易被听到，也容易被写成干扰项。

盲推策略：

- 如果只有一个选项像具体例子，其他选项更抽象，先降权例子项。
- 如果四个选项都是具体例子，那就不能用这条。

## 规则 8：抽象项要分完整概括和泛化包装

425 题中：

- 唯一抽象项（宽口径抽象核心）：`62` 次出现，命中 `18` 次，约 `29.0%`；A/B/C 分别是 `3/13`、`6/20`、`9/29`。
- 泛化抽象词（impact/effect/value/role/quality/importance/significance 这类）有 `39` 次出现，命中 `8` 次，约 `20.5%`。

抽象项常含：

- `issue`
- `problem`
- `purpose`
- `impact`
- `effect`
- `risk`
- `benefit`
- `relationship`
- `correlation`
- `attitude`
- `confidence`
- `productivity`
- `satisfaction`
- `anxiety`
- `welfare`
- `strategy`

为什么有时有用：

六级听力，尤其 Section B/C，经常考“这段说明什么”“研究发现什么”“讲话者想表达什么”。但这不等于任何抽象词都加分，只有能把关系说完整的抽象概括才有用。

盲推策略：

- 主旨题、结论题、讲座题，完整概括型抽象项可以提高优先级。
- 细节题中，唯一抽象项可能范围过大，不能盲选。
- 这里说的“抽象项”必须是能概括原文关系的整句，不是只含 `impact/effect/value/role/quality` 这类泛化名词。

## 规则 9：信号词要分“答案味”和“包装味”

同样是高级词，有的更像答案，有的更像干扰。宽松定义下的统计显示：

- 评价词，如 `successful/curious/positive/anxiety/worth`：`13/38`，Section C 里是 `10/25`，只有正好回答态度/结论时才加分。
- 未来/计划/预期，如 `future/plan/expect/improve/expand`：`6/24`；2025 中，Section A 末题 future/plan 是 `0/3`，所以只能在对话链已经走到下一步时加权。
- 研究/data/study：`1/17`，说明它不能当自动答案。
- 泛化抽象词，如 `impact/effect/ability/value/role/quality/importance/significance`：`8/39`，更像包装。
- 具体人名/地名/机构名：`4/16`；如果只是一个例子而没有关系链，仍然先降权。
- `one of / one of the most...`：选项里只出现 `1` 次，命中 `0` 次，不能写成规律；它更像“最高级包装/身份标签”，不是直选信号。
- `cause/reason/because/result`：`2/16`，更像“关系提醒”，不是硬排除。
- `problem/challenge/concern/risk`：`6/23`，要看它是不是在回答题问，而不是把语气说重。
- 数字：`10/48`，还是更常见于背景信息，而不是题眼本身。

这组结论最重要的不是具体百分比，而是分类：

1. **答案味信号**：评价、态度、关系延续、可转述成一句结论的表达；未来词只有接住上下文时才算。
2. **包装味信号**：研究壳、数字、泛化抽象名词、过具体专名、`one of the most...` 这类最高级身份包装。
3. **关系提醒信号**：因果词、问题词、比较级、弱语气。它们不宜直接选，也不宜直接排，更像提醒你先判断题目到底在问什么。

考场判断：

- 评价词如果正好回答“怎么看某人/某事”，可以加分；否则只是态度壳。
- `impact of...`、`ability to...`、`value of...` 如果没有说清“谁影响谁、怎么影响”，不要因为抽象就选。
- `research/data/study` 如果只有一个选项有，先怀疑；如果三四个选项都有，说明题目本来就在问研究发现，这时不要排整个研究簇。

## 规则 10：唯一未来/计划项只能低权重使用

425 题中：

- 宽松定义下的未来/计划项：`24` 次出现，命中 `6` 次，约 `25.0%`。
- 严格“唯一 future”口径下：`16` 次出现，命中 `3` 次，约 `18.8%`；A 末题 future/plan 规则是 `1/6`。
- 2025 中，A 的 Q4/Q8 若只看唯一未来/计划候选，是 `0/3`。

这说明它不是稳定蒙题公式。它有用的前提是：前几题已经把对话或讲座推到“下一步/后续行动/长期结果”；否则命题人很容易把 `next/future/plan/advice` 写成显眼干扰项。

- Section A 末题常问 future plan。
- 选项中只有一个明显含 `future/will/plan/intend/add/expand/open`，只能说明它值得进入候选，不等于值得直接选。

盲推策略：

- Q4/Q8 先问：前面几题有没有把话题推到“决定/下一步/后续行动”？
- 如果没有这条推进线，唯一未来/计划项先降权。
- 如果这个未来项过度具体，而另一个选项是上位概括，要优先概括项。

## 规则 11：相反项不是答案保证，而是二选一预警

425 题里，严格可自动识别的相反项不多，因此不能给出稳定命中率；但 `positive/negative` 只抓到 `4` 题，`more/less` 只抓到 `8` 题，它们仍然更像“二选一预警”，不是答案保证。

> 相反项的最大价值不是告诉你选哪个，而是告诉你题目会考哪个维度。

常见相反维度：

- positive vs negative
- more vs less
- can vs cannot
- effective vs ineffective
- accept vs reject
- theory benefit vs actual risk
- speaker A attitude vs speaker B attitude

盲推策略：

1. 如果两个选项同维度正反相对，先把它们都保留。
2. 如果另两个选项只是例子/背景/范围过大，优先排除。
3. 最后在相反项中，看哪个更符合题号位置：
   - Section A 态度题：跟题问说话人。
   - Section B 研究题：跟研究结论。
   - Section C 结尾题：跟总结/现实风险。
4. 如果这一组里还能看到一对表述相近的选项，先把它们当候选簇；相反项负责告诉你方向，相近项负责告诉你答案可能落在哪一簇。

## 规则 12：相近项更像候选簇，尤其在 Section C

这条比相反项更有挖掘价值。

这里说的“相近”不是“意思差不多就行”，而是“表面框架接近”。

脚本里的相近度不是语义模型，而是把 4 个表面信号加权起来：

1. **内容词重合**。先去掉 `a/the/of/and/is/are` 这类虚词，只留下 `research/problem/result/benefit/friendship` 这类内容词，再看两项共享了多少内容词。
2. **内容词 Dice**。还是看内容词集合，但更偏向“共有词越多，分越高”。
3. **字符 3-gram 重合**。把句子转成小写，去掉非字母字符，切成连续 3 个字符的小片段，看这些片段重合多少。这个会抓到很多“句面长得像”的选项。
4. **整句字面顺序相似度**。看词序和句型是不是也接近。

脚本里的总分大致是 `45% 内容词重合 + 25% 内容词 Dice + 20% 字符 3-gram + 10% 整句顺序相似度`。

所以它其实在找的是：

- 共享同一批关键词的两项。
- 共享同一个句式骨架的两项。
- 一个是另一个的压缩改写或轻微换词版。

它不懂深义，也不负责判断同义词是不是完全等价，所以考场上不要把它当“精确计算题”。你只要把它当成“先把最像同一簇的两个选出来”的工具。

425 题里：

- 最相近对分数 `>=0.20`：`160` 题，答案在这对里 `108` 题，约 `67.5%`，高于随机保留两项的 `50%`。
- 分数 `>=0.22`：`136` 题，答案在这对里 `92` 题，约 `67.6%`。
- 分数 `>=0.24`：`108` 题，答案在这对里 `73` 题，约 `67.6%`。
- 分数 `>=0.26`：`86` 题，答案在这对里 `59` 题，约 `68.6%`。
- 分数 `>=0.28`：`65` 题，答案在这对里 `43` 题，约 `66.2%`。
- 分数 `>=0.30`：`54` 题，答案在这对里 `37` 题，约 `68.5%`。
- 分数 `>=0.32`：`40` 题，答案在这对里 `29` 题，约 `72.5%`，但题数已经很少。

按 section 看，差异很明显：

- Section A：相近对 `>=0.20` 时是 `31/47`，约 `66.0%`；只能当弱候选。
- Section B：相近对 `>=0.20` 时是 `22/38`，约 `57.9%`；`>=0.24` 时是 `13/23`，约 `56.5%`，仍不稳定。
- Section C：相近对 `>=0.20` 时是 `55/75`，约 `73.3%`；`>=0.24` 时是 `39/53`，约 `73.6%`；`>=0.28` 时是 `27/34`，约 `79.4%`。

这说明：

1. **相近项不是答案保证，但常是候选簇**。特别是 Section C，两个表述相近的选项往往围绕同一个研究发现、机制或结论。
2. **相近对里更短、更压缩的项更值得看**。在分数 `>=0.24` 且答案落在相近对里的 `73` 题中，答案比另一个相近项更短的有 `47` 题，约 `64.4%`；Section C 中约是 `30/39`，约 `76.9%`。
3. **分数越高，Section C 的短项倾向越明显**。相近对 `>=0.28` 时，答案在 C 的相近对里是 `27/34`；其中答案更短约是 `21/27`。
4. **相近对也可能是一组共同干扰**。如果两个相近选项都只是例子、都只是正面包装、都没有回答题问，答案可能在外面。

考场手判版，可以这样做：

1. **先去虚词**。把 `a/the/of/and/is/are/to` 这类词先从脑子里删掉，只看剩下的内容词。
2. **抓骨架**。看两项是不是同一个“谁 + 做什么 + 对什么 + 得到什么结果”的骨架。
3. **看是不是同一方向**。比如都是“X 导致 Y”或都是“X 帮助 Y”，还是一个正向一个反向。
4. **看是不是一个是另一个的缩写版**。更短的往往不是因为信息少，而是因为把同一关系压缩了。
5. **看对象有没有偷换**。如果两项共享很多词，但一个说人与人、另一个说人和制度，表面像，实质未必同簇。

你可以把它记成三问：

- **同对象吗**？
- **同关系吗**？
- **只是更短的同义改写吗**？

如果三问都差不多是“是”，就把它们当候选簇。

考场上可以直接分三档：

- **高相近**：去掉虚词后，至少共享 2 个以上内容词，且句骨架也像。比如都在说同一对象、同一动作、同一结果。
- **中相近**：只共享 1 个核心词，但对象和关系方向也差不多，像是同一概念的换说法。这种在 Section C 里可以先留着，在 Section A/B 里别太激进。
- **低相近**：只共享一个大而空的词，比如 `important/benefit/role`，或者只是同一话题但关系完全不同。这种不要当候选簇。

不算相近的常见情况：

- 只共享一个大词，比如都带 `important`、`benefit`、`role`，但前后关系完全不同。
- 只共享一个主题词，比如都在说 `students`、`research`、`health`，但一个问原因一个问结果。
- 只是语气都偏正面，但没有同一条逻辑链。

典型“答案在相近对里”的情况：

- A. `They are quite friendly to humans`
- C. `They are unafraid of humans`

这两项都围绕“人与熊的关系”，答案是 C。C 更短，也更像原文结论的压缩表达。

再看 Section C 的一组：

- B. `Working together enhances friendship`
- D. `Friendships benefit work`

这两项互相接近，但 D 更像最终结论，答案是 D。这里不是选“更长”，而是选“关系方向更贴合题问”的那一个。

典型“相近对是共同干扰”的情况：

- A. `They bring more benefits to young people`
- C. `They are more suitable to young people`
- D. `They have fewer rules and pressures`

A/C 很像，都是“对年轻人更好”的正面包装；答案却是 D，因为题点不是泛泛说好，而是说明规则和压力更少。

盲推策略：

1. 看到两项表述很像，先不要把它们互相抵消；先把它们当成同一候选簇。
2. Section C 中，如果相近对围绕同一发现、机制、影响，优先在这对里比较。
3. 在相近对内部，优先看更短、更压缩、关系方向更清楚的项。
4. 如果相近对只是同一类包装词，比如都在说 `benefit/suitable/important`，但没有回答题问，就把这对一起降权。
5. Section B 不要过度使用这条；短文题更常用相近表述制造同一维度的干扰，仍然要回到段落功能。
6. 如果你完全没法判断，就先把这对记成“候选簇”，暂时别把外面的单独选项看得比它们更强。

一句话：

> 相近项的价值不是“二选一必中”，而是帮你看出命题人围绕哪个概念设了候选簇；在 Section C 里，这个候选簇尤其值得保留。

## 规则 13：题号位置有弱先验，但不能押字母

有些题号出现弱分布：

- Q1：A 较多，7/17。
- Q5：A/D 并列最强，都是 5/17。
- Q8：B/D 并列最强，都是 6/17。
- Q10：B 较多，7/17。
- Q18：B 较多，6/17。
- Q24：D 较多，8/17。
- Q25：B/C 并列最强，都是 5/17。

但题数少，且考试不可能稳定押字母。

正确用法：

- 题号先验只能在完全无信息、两三个选项都无法判断时作为最后一层。
- 不要为了押分布推翻更强的选项结构判断。

## 规则 14：整套字母均衡和连号只能兜底

在 17 套里，单套听力答案分布通常比较均衡；相邻题更常切换字母，而不是一直连号。

观察到的 17 套：

- `2022.06.set1`：A/B/C/D 分布 `7/6/5/7`，最长连号 2。
- `2024.06.set2`：A/B/C/D 分布 `7/5/6/7`，最长连号 4，是最明显的连号例外。
- `2025.06.set2`：A/B/C/D 分布 `6/6/6/7`，最长连号 1。
- `2025.12.set2`：A/B/C/D 分布 `6/6/7/6`，最长连号 2。
- 其余各套也都落在 `6/6/6/7` 附近，没有极端偏置。

用法：

- 相邻两题题号连续的统计中，同字母只有 `55/408`，约 `13.5%`。
- 连续三题同字母是 `2/391`，约 `0.5%`。
- 如果前两题已经同字母，第三题继续同字母是 `2/55`，约 `3.6%`。
- 17 套里最长连号只有 4，而且只出现在 `2024.06.set2`。
- 大多数套卷都没有极端偏置，所以不要用字母均衡替代选项逻辑。

禁忌：

- 不要为了“均衡”改掉有把握的答案。
- 不要看到三个题里缺某个字母，就硬补那个字母。
- 不要看到两个 A 后就断定第三个不可能是 A，只能在完全无信息时降权。
- 不要用字母分布替代选项逻辑。

这条规则只能放在最后一层，优先级低于选项结构、题型、听到的任何关键词。

最实用的一句话：

> 字母均衡不是“缺什么补什么”，而是“已经二连时别轻易追第三个同字母”。

## 多信号叠加评分

单条蒙题经验都很悬，真正能提高一点胜率的是叠加。可以把每个选项当成一个候选人来加减分。

强排除信号：

1. 字符唯一最长，同时字母唯一最多。
2. 又长又有唯一数字、唯一研究词、唯一因果词。
3. 选项只是在说背景、例子、身份、专名，没有回答题问动作。
4. 绝对词特别满，如 `all/only/never/always`，但题干不像在问绝对判断。

弱加分信号：

1. 字母数少、字符短，而且关系完整。
2. Section A 末题出现未来、下一步、建议、计划，但要先看对话链是否已经推到那一步。
3. Section B 里能说明段落功能的选项，比只复述例子更像答案。
4. Section C 里能说清发现、机制、影响的选项，比只有学术词的项更像答案。
5. 前后题能串成同一条链，后题选项正好接住上一题锚点。

最后裁判信号：

1. 题号弱先验，如 Q1 偏 A、Q5 A/D 并列、Q8 B/D 并列、Q10 偏 B、Q24 偏 D、Q25 B/C 并列。
2. 已经出现二连同字母时，第三题完全没信息就少追同字母。
3. 四题组不要只压两种字母，但也不要机械补缺失字母。

考场上不要算复杂分数，只做四步：

1. 先定 section：A 看对话推进，B 看段落功能，C 看概念关系。
2. 再排掉本 section 里的强排除项。
3. 再看谁拿到本 section 的专属弱加分。
4. 仍然五五开，才用题号和字母均衡做最后一票。

## 2025 数据表现

2025.06 和 2025.12 的第一、第二套共 100 题表现如下：

1. **字母/字符最少仍然能用，但只是弱加分**。2025 里字符唯一最短命中 `41/89`，字母唯一最少也是 `41/89`，约 `46.1%`；Section A 的字符最短/字母最少分别是 `15/28`、`14/27`，Section B 是 `10/24`、`10/24`，Section C 是 `16/37`、`17/38`。
2. **又长又字母多仍然适合强排除**。字符最长且字母最多的候选里，2025 只命中 `5/76`，约 `6.6%`；放到 Section C 里是 `0/27`。
3. **比较/研究/因果/绝对/未来/例子这些词，2025 里不能当硬规则**。比较 `0/6`、研究 `0/5`、因果 `0/3`、绝对 `0/4`、future `0/4`、例子 `0/2`，更像“先检查关系”的标记，而不是直接押的依据。
4. **唯一否定项不再是稳定坑**。2025 里它反而有 `5/8` 的命中，说明否定词只能当逻辑轴提醒，不能自动排，也不能自动选。
5. **Section A 的末题未来/计划不该再机械押**。2025 里和 `future/plan/advice` 相关的唯一候选在 A 的 Q4/Q8 上是 `0/3`，更像干扰包装；先把对话链接住，再考虑这类词。
6. **字母分布本身仍然很均衡**。2025.06.set1 是 `7/6/6/6`，2025.06.set2 是 `6/6/6/7`，2025.12.set1 是 `6/6/6/7`，2025.12.set2 是 `6/6/7/6`；最长连号分别是 2、1、2、2，说明不能靠整体押字母。

## 盲推排除清单

如果完全没听到，先尝试排除这些：

1. **字符最长且字母最多的堆叠选项**。
2. **唯一显式因果词项**，除非四个选项整体都像原因题。
3. **唯一数字项**，除非四个选项都在比数字。
4. **唯一研究词项**，如果它只说 study/research 本身，没有具体发现。
5. **唯一具名例子项**，尤其在 Section C。
6. **唯一过强绝对项**，如 `all/only/never/always`，除非原文可能明确绝对。
7. **泛化抽象包装项**，如只说 `impact/effect/value/role/quality`，但没有完整关系。
8. **唯一否定项**，不要自动排，也不要自动选；先看它是不是在回答题目本身。
9. **相近但共同包装的两项**，如果都只是 `benefit/suitable/important` 这类泛泛好话，先把这一簇一起降权。

## 盲推选择清单

排除后，如果还剩多个选项，可优先考虑：

1. **简短、直接、关系完整的选项**。
2. **字母数少、字符短，但不是孤立名词的选项**。
3. **唯一抽象概括项**，尤其主旨/结论/讲座题。
4. **评价/态度/情绪词能直接回答题问的选项**。
5. **唯一未来/计划项**，只在 Section A 的对话链已经推到下一步时才加权，不能看到 Q4/Q8 就先押。
6. **相反项中的更合理一端**，但不要跳过题问对象。
7. **相近候选簇中的短而完整项**，尤其 Section C。
8. **没有过强限定、没有对象偷换、没有例子化的选项**。

## 分 section 盲推策略

### Section A

Section A 是两段长对话，最适合用“对话链”蒙题。它不是讲座式展开，而是两个人围绕一个具体问题来回推进。

Section A 有 136 题。短项倾向存在，但不是网传那种“数几个字母就能押答案”。Section A 里选“字母唯一最少”是 `51/112`，约 `45.5%`；而“字符最长且字母最多”只命中 `11/108`，约 `10.2%`。
2025 中，Section A 的唯一字符最少/字母最少分别是 `15/28`、`14/27`，但末题里的未来/计划单独加分是 `0/3`，说明它只能当最后一层，不能替代对话链。

题位推理：

1. `Q1/Q5` 常先定人物身份、当前状态、谈话主题。
2. `Q2/Q6` 常问评价、原因、问题、早期经历。
3. `Q3/Q7` 常问做法、态度、坚持的理由。
4. `Q4/Q8` 常落到建议、决定、下一步、未来计划。

Section A 的加分项：

1. 短而完整：例如 `staff writer`、`mostly failures`、`follow the man's advice` 这种能直接回答题问。
2. 能接住前题：前面讲创业原因，后面就更可能问扩展业务；前面讲论文建议，后面就更可能问是否采纳建议。
3. 末题里的未来/计划/建议：尤其 `will/would/plan/follow/advice/expand/continue`。
4. 评价词如果正好回答“怎么看”：如 `curious and autonomous` 这类不是空泛高级词，而是人物评价。
5. `problem/failure/debt` 这类负面词在 Section A 不一定是坑；如果它描述人物当前困境或早期经历，反而可能是题点。

Section A 的排除项：

1. 最长且字母最多，尤其还塞了家庭背景、身份背景、附加限定。
2. 把另一个说话人的立场套到题问对象上。
3. 只像身份标签，但没有回答题问动作，比如“某人是什么职业”不一定等于本题答案。
4. 否定词、绝对词不要直接选；它们更像提醒你判断说话人态度。
5. Section C 那套“研究发现/概念链”不要硬套到 Section A；对话题更常考谁说了什么、下一步怎么做。

Section A 的专用顺序：

1. 先判定题问对象是男士、女士，还是第三人。
2. 再把本组前题压成一条人物线：身份 -> 问题 -> 原因/做法 -> 下一步。
3. 最后才用短项、字母数、未来词加权。

盲推建议：

> Section A 的核心不是押字母，而是“人物关系 + 前后推进”。完全没听到时，先找短而完整的状态/动作项，再看它能不能接住前一题。

### Section B

Section B 是短文/篇章题，比 Section A 少了对话轮次，更多靠段落功能：开头立题，中间解释，结尾收束。

Section B 有 119 题，所有表面规律都要让位给段落功能。选“字母唯一最少”是 `36/98`，约 `36.7%`；“字符最长且字母最多”命中 `13/97`，约 `13.4%`。
2025 中，比较/研究/因果/绝对这几类词继续更像排坑项，但唯一否定项在 B 里是 `4/5`，已经不能自动当坑。

题位推理：

1. `Q9/Q12` 常问主题、现象、故事开端、研究对象。
2. `Q10/Q13/Q14` 常问原因、功能、过程、例证。
3. `Q11/Q15` 常问结论、启示、建议、结果。

Section B 的加分项：

1. 能概括整段功能的选项，比局部细节更值得保留。
2. 结尾题看建议/结果，不要被中间例子带跑。
3. 如果题目像 Why 题，不要机械选含 `cause/reason` 的选项，要选真正解释“功能/目的”的项。
4. 多个选项都带研究/data 味时，答案通常在这个候选组里；此时看谁说的是发现本身。
5. `future/avoid/help/reduce` 这类词如果表达段落目的或建议，比单纯“未来计划”更有用。

Section B 的排除项：

1. 唯一数字项、唯一因果词项、唯一研究词项，若只是显眼包装，先降权。
2. 只复述一个例子、一个场景、一个工具，但没有上升到段落功能。
3. 过强结论，如 `must/always/only`，除非四个选项都在比强弱。
4. 长而泛的抽象项，如只说 `impact/effect/value/ability`，但没有清楚对象关系。
5. Section A 的“谁说话/下一步”不要硬套到 Section B；短文题更少问对话回合，更多问这段话想说明什么。

Section B 的专用顺序：

1. 先判断这题在段落里的位置：开头立题、中间解释、结尾收束。
2. 再判断题问的是主题、原因、过程、例子、结果，还是建议。
3. 最后用显眼词排坑：数字、cause、research、泛化抽象词只作降权，不作直选。

盲推建议：

> Section B 的核心是“段落功能”。完全没听到时，先问：这段是在解释原因、说明现象、讲一个结果，还是给建议？

### Section C

Section C 是记录/讲座题，最适合用“研究发现或概念链”蒙题。这里的选项常常都很抽象，所以不能只看短长，要看关系是否完整。

Section C 有 170 题。“字符最长且字母最多”只命中 `7/128`，约 `5.5%`，适合做强排除；字母唯一最少是 `70/150`，约 `46.7%`。更值得看的其实是相近候选簇：相近对分数 `>=0.24` 时，Section C 答案在这对里是 `39/53`，约 `73.6%`。
2025 中，这个“又长又字母多”的排除感更明显，字符最长且字母最多在 C 里是 `0/27`；但抽象词只有拼得出完整发现/机制/影响链条时才算加分。

题位推理：

1. `Q16-Q18` 常先问研究/观点/概念，再问发现、机制或影响。
2. `Q19-Q21` 常围绕同一个概念展开定义、原因、结果。
3. `Q22-Q25` 常是最长的一组，后两题更容易问启示、应用、总体特征。

Section C 的加分项：

1. 评价/态度/情绪词能概括发现：如 `anxiety affects...`、`worth/value` 但必须有完整关系。
2. 未来/计划/长期结果：讲座结尾常从发现推到意义。
3. 比较级、否定词、弱语气如果多项同现，先当成候选组，不要单独选或单独排。
4. 研究/data/study 如果多项同现，说明题目在问研究内容；保留说“发现本身”的选项。
5. 因果词在 Section C 只有说清“机制链”时才加分；单独一个 `result/cause` 仍然像包装。

Section C 的排除项：

1. 超具体人名、地名、机构名，尤其在主旨/结论题里。
2. 唯一研究词但只说研究方法、数据、survey 本身，没有发现。
3. 唯一问题词，如 `problem/risk/challenge/concern`，如果它只是把语气说重。
4. 泛化抽象包装：`impact/effect/role/quality/value` 后面没有明确对象关系。
5. Section A 的“短而直接”不能原样搬来；Section C 的正确项可以短，但必须是一个完整概念，不是一个标签。

Section C 的专用顺序：

1. 先找本题问的是发现、机制、影响、例子，还是总结。
2. 再看选项有没有完整关系：A 导致 B、A 影响 B、A 依赖 B。
3. 最后用具体名词、研究壳、泛化抽象词排除只像局部碎片的选项。

盲推建议：

> Section C 的核心是“概念关系”。完全没听到时，不要选最像学术词堆的项，要选最能概括“发现/机制/影响”的项。

## 前后题串联盲推

这是最容易被忽略、但最有用的一层。

如果前一题你已经听到了，后面题没听到，不要把它们当独立题。六级同一段音频里的后面几题，通常在同一条逻辑链上，只是换了角度问。

### 这层规则怎么用

1. 把前题答案压缩成 1-2 个锚点词。
2. 问自己：后题是在问同一对象的什么新属性？
3. 看后题选项里，哪个最符合这个锚点所建立的主题。
4. 把和锚点冲突的选项先排掉。
5. 如果后题完全没听到，就把它当“同主题的下一步推演”，而不是重新从零猜。

### Section A 的串联方式

Section A 最强的是 1-4、5-8 这两组对话链。

#### 1-4 的典型链

常见顺序是：

1. 介绍人物/身份/场景
2. 描述对象特点/当前状态
3. 解释行为原因或帮忙方式
4. 回到个人感受、成长变化或总结判断

所以如果你已经知道：

- `Q1` 是谁
- `Q2` 这个人/群体是什么状态
- `Q3` 做法是什么

那 `Q4` 通常不会跳到别的话题，而是继续问“个人变化/态度/总结”。

#### 5-8 的典型链

常见顺序是：

1. 介绍生意/项目/工作
2. 追问起点或初始情况
3. 追问坚持的原因或核心困难
4. 问未来计划或扩张方向

所以如果 `Q5` 已经告诉你人物身份，`Q6` 告诉你一开始很失败，`Q7` 往往会问为什么能坚持，`Q8` 往往会问未来怎么发展。

### Section B 的串联方式

Section B 一般是“主题 -> 证据 -> 机制 -> 结论/建议”。

如果你已经听到前一道题的答案，后一道题常常不是新话题，而是同一话题的下一层：

- 先问研究在说什么
- 再问为什么重要
- 再问具体发现
- 再问推论或建议

所以：

- 你若知道前题是“AI 找新抗生素”，后题就大概率围绕耐药性、化合物毒性、筛选效率这些主题。
- 你若知道前题是“短期压力有益”，后题就大概率绕回短期/长期、免疫、疾病风险这些限定。

### Section C 的串联方式

Section C 更像总-分-总。

你一旦知道前题的主旨，后题通常只是在这个主旨上展开：

- 开头先给问题背景或目的
- 中间问具体建议/分类依据/实验步骤
- 后面问推论、总体特征、长期影响

例如：

- 先知道讲的是医院不尊重病人，后面题就大概率围绕 advocate、记录姓名、被动患者这些点。
- 先知道讲的是团队 vitality，后面题就大概率围绕 similarity、diversity、future collaboration。
- 先知道讲的是节俭和财富，后面题就大概率围绕 confidence、long-term investment、keep up with others。

### 串联时最实用的排除法

如果前题你已经知道答案，可以先问下面 4 个问题：

1. 后题是不是在问同一个人物/对象？
2. 后题是不是在问同一段关系里的下一步？
3. 后题是不是在问前题答案的原因、后果、例子或总结？
4. 后题选项里谁最符合前题建立的主题锚点？

### 串联的具体例子

#### 例子 1：2022.06 set1 Q1-Q4

- `Q1`：David Jackson 是 staff writer。
- `Q2`：teenagers 独立、好奇。
- `Q3`：对写作用 encouragement，而不是 editorial guidance。
- `Q4`：自己十几岁时书的品味改变了。

这条链说明这组对话一直在讲“孩子、阅读、写作、青春期变化”。  
如果你只听到前面两题，后面题没听到，也别去选和孩子/阅读无关的项。

#### 例子 2：2022.06 set1 Q5-Q8

- `Q5`：女方是大学生，也是成功年轻企业家。
- `Q6`：她刚开始做衣服时大多是灾难。
- `Q7`：她坚持是因为自己身材高、买不到合适衣服。
- `Q8`：未来继续 online，并扩展儿童装/男装。

这条链的核心锚点是“自己需求推动创业，后来扩张业务”。  
如果后题没听到，看到“开实体店”“改做别的行业”“不再做服装”这类选项，基本都可排。

#### 例子 3：2022.06 set1 Q19-Q21

- `Q19`：vitality 来自成员解散后还保持联系。
- `Q20`：相似性促进未来合作。
- `Q21`：当前团队是多样化但最成功、最有凝聚力的。

这条链告诉你，这段在讲“团队关系如何形成并维持”。  
所以后题如果出现“完全无关的管理话题”就应优先排掉。

#### 例子 4：2022.06 set1 Q22-Q25

- `Q22`：节俭 = 存 20%，花 80%。
- `Q23`：信心帮助专注长期投资。
- `Q24`：百万富翁失败时不怪别人。
- `Q25`：他们不会跟别人攀比。

这条链说明整段在讲“财富心态的自律和长期性”。  
如果你只记住 `save 20%`，后面题没听到，也可以优先排除那些讲短期享乐、跟风消费、把失败归咎他人的选项。

### 一句话版

> 前题不是独立信息，而是后题的主题锚点。后题没听到时，先拿前题答案建立同一段的逻辑主线，再用这条主线去排除不一致的选项。

## 盲推流程

完全没听到时，按这个顺序来：

1. 先定 section：A 是对话推进，B 是段落功能，C 是概念关系。
2. 再看题位：A 的前中后、B 的开头/中间/结尾、C 的发现/机制/总结。
3. 按本 section 的方法先排跑题项：A 排对象偷换，B 排局部例子，C 排研究壳和具体名词。
4. 再看通用强排除：字符最长且字母最多、唯一数字、唯一因果词、泛化抽象包装。
5. 再看本 section 的加分项：A 看短而完整和下一步，B 看段落功能，C 看发现/机制/影响。
6. 如果前后题能串联，用上一题锚点继续排除跑题项。
7. 最后才用题号/section 字母弱先验和二连不追三连。

## 盲推决策树

如果要把上面的经验压成一个考场可执行版本，可以这样走：

1. **先分区**：A 用人物/对话链，B 用段落功能，C 用概念/研究链。
2. **再问题型**：身份/行为/原因/建议/主旨/结论/态度。
3. **先排本区跑题项**：A 排对象偷换，B 排局部例子，C 排学术词壳。
4. **再排最长包装项**：特别是字符最长、字母最多、信息堆叠、范围很大的选项。
5. **排唯一显眼诱饵**：唯一数字、唯一研究词、唯一因果词、唯一具名例子、泛化抽象包装项。
6. **保留同维度相反项**：如果两个选项正反相对，先别排，等最后二选一。
7. **按本区加分**：A 看短而完整和下一步，B 看功能/目的/结果，C 看发现/机制/影响。
8. **仍无法判断时看均衡**：避免连续第三个同字母，四题组尽量不要只压两种字母。

这个流程的本质是：先判断这道题属于哪种听力结构，再排掉这个结构里最常见的诱饵。

## 盲推口诀

最长多半是包装，最短有时是答案；  
数字研究别冲动，因果显眼常设坑；  
例子不像主旨，抽象更像结论；  
相反项定维度，未来项看末题；  
完全没听到，先排诱饵，再选简洁概括。

## 最重要的底线

盲推策略只能把随机 25% 往上抬一点，不可能替代听原文。一个简单组合表面评分模型也只是比随机好一点，说明“只看选项”确实有信息，但远远不够稳。

因此最佳用法是：

- 听到一点：用这些规则做排除和补救。
- 完全没听到：用这些规则提高盲猜质量。
- 能听到题点句：永远以题点句逻辑为准。

## 老师版例题

下面这一节不再讲空规则，直接拿真题来拆。你可以把它当成“盲推时的现场演示”。

### 1. 最长项常常只是包装

#### 2022.06.set1 Q1

- A 是 `staff writer`，D 是 `father of four kids`。
- 先盲推时，D 看起来信息更多，但它其实把身份后面又塞了一层家庭背景。
- 正确答案是 A。

这题的教训很直白：最长项不一定更完整，很多时候只是更会包装。

#### 2022.06.set1 Q20

- B 是“相似性有助于未来合作”，D 是“互相理解来自共同目标”。
- 两个都像研究结论，但原文真正落点是“相似性 -> 未来合作”。
- 正确答案是 B。

这类题特别像命题人给你两句都很像样的话，真正要听的是哪一句更贴题。

### 2. 最短项如果关系完整，就很能打

#### 2022.06.set1 Q4

- A 是“书的品位变了”，四个选项里最短。
- 它不是偷懒，而是刚好把题点说完了。
- 正确答案是 A。

#### 2022.06.set1 Q6

- D 是“刚开始大多是失败”。
- 这句话短，但它把“早期做衣服的结果”说透了。
- 正确答案是 D。

#### 2022.12.set1 Q1

- A 是“还没开始写论文”。
- 这也是典型的短而直接，正好命中开头状态。
- 正确答案是 A。

所以短项不是天然优势，短而完整才有用。

### 3. 绝对词和否定词，不能一刀切

#### 2024.12.set1 Q14

- A 是“never repeated”。
- 这题里绝对词不是陷阱，因为原文就是“没有两次会一样”。
- 正确答案是 A。

#### 2022.12.set2 Q17

- A 是“strictly forbidden”，D 是“only in certain areas allowed”。
- 这种选项属于同一限制轴上的强弱对照，不能看到一个绝对词就立刻选或立刻排。
- 正确答案是 A。

结论是：绝对词只是在提醒你去核原文，不是自动判死刑。

### 4. 研究词和 data 词，不等于答案

#### 2023.12.set2 Q16

- B 看着很学术，像是在争论 survey data。
- 但题目真正问的是“他们要说明什么”，答案是 A，和研究壳子不完全是一回事。

#### 2024.12.set1 Q17

- 研究目的不是去证明“左/右谁更对”，而是看不同社会有没有不同版本的道德观。
- 答案是 B。

所以看到 `research/study/data/experiment`，先问自己：它是在说壳，还是在说发现。

### 5. Section A 末题经常是“未来/下一步”

#### 2022.06.set1 Q5-Q8

- Q5 先把女方定成“大学生 + 年轻创业者”。
- Q6 说她最开始做衣服时大多失败。
- Q7 说她为什么坚持，是因为自己身材高，买不到合身衣服。
- Q8 就顺势落到扩展业务。

这是一条很标准的对话链：身份 -> 起点 -> 原因 -> 未来。

#### 2022.12.set1 Q1-Q4

- Q1 论文还没写。
- Q2 教授给了找资料的建议。
- Q3 提醒题目不够大。
- Q4 最后就变成“follow the man's advice”。

你会发现，后面的题不是重新起炉灶，而是前面几题往下推一步。

#### 2022.12.set2 Q1-Q4

- 先讲“幸福比钱更重要”。
- 再讲成为作家前的债务和牺牲。
- 最后落到“为了理念要做重大取舍”。

这类题只要听到主题，就能把后面两题的方向先锁住。

### 6. 同维度二选一，先别急着赌

#### 2024.12.set1 Q16

- C 是“所有社会由七条道德规则维系”。
- D 是“所有人都共享同一种普世道德标准”。
- 这两个几乎在同一个轴上，但 D 更绝对、更泛化，原文却是 C。

这就是很典型的“强版本看着爽，但原文只到中间那层”。

#### 2022.12.set2 Q19

- B 是“没有足够知识就做判断”。
- D 是“被证明错了还坚持判断”。
- 原文是在定义 prejudice，答案落在 B。

这题告诉你：同维度选项别只看谁更狠，要看谁更像定义句。

### 7. 前后题串联，最值钱

#### 2022.06.set1 Q19-Q21

- Q19: 团队 vitality 来自成员散了以后还保持联系。
- Q20: 成员相似性有助于未来合作。
- Q21: 当前团队是多样化的，但最成功、最有凝聚力。

这三题其实都在同一条线上：团队关系怎么形成、怎么维持、怎么扩展。
如果你已经听到 Q19，Q20 和 Q21 就不该当成新话题。

#### 2022.06.set1 Q22-Q25

- Q22: 先存 20%，再用剩下的生活。
- Q23: 信心帮助人专注长期投资。
- Q24: 百万富翁失败时不甩锅给别人。
- Q25: 他们不试图跟别人保持一致。

这组就是“节俭 + 长期主义 + 不跟风”的整条链。
只要前两题听到了，后两题就能顺着推。

#### 2023.06.set2 Q12-Q15

- Q12: 新产品很快变成垃圾，因为科技公司故意让它们寿命短、难维修。
- Q14: 所以技术公司要带头减少电子垃圾。
- Q15: 这个问题是可以解决的。

这组的主线很清楚：问题是人为造成的，所以也就不是无解的。

#### 2024.12.set1 Q9-Q11

- Q9: 选带路牌的照片，是为了让电脑确认你是人。
- Q10: 研究目的是找出我们为什么会忘密码，好帮助记忆。
- Q11: 网站应该多给激励，让用户更常登录。

这组的核心不是“照片”，而是“记忆密码的机制”。

#### 2024.12.set2 Q22-Q25

- Q22: 气味能触发情绪记忆。
- Q23: 关键在气味分子和大脑相关区域的互动。
- Q24: 回忆受 context 影响。
- Q25: 气味记忆既个人化，又会变化。

这组特别适合练“前题当锚点，后题沿锚点推”。

### 8. 三个 Section 的不同打法

#### Section A：先看说话人和对话推进

`2022.06.set1 Q2`

- A 是 `interested in fairy tales`，只是阅读内容的局部。
- B 是 `curious and autonomous`，正好回答“男士怎么看青少年”。
- C 是 `headache to their parents`，情绪太负。
- D 是 `ignorant of politics`，话题偷换。
- 正确答案是 B。

这题不能只看短长。Section A 问“怎么看某人”时，评价词如果刚好概括说话人态度，反而要保留。

#### Section B：按段落功能，不按显眼词

`2022.12.set1 Q10`

- A 是 `To avoid them in the future`。
- D 是 `To reflect on their causes`。
- 题目是 Why，但不是看到 `causes` 就选 D。
- 正确答案是 A，因为短文真正解释的是“危险记忆的功能”：以后避开危险。

Section B 的 Why 题要问“这段在解释什么功能”，不是机械找 `reason/cause`。

`2023.06.set2 Q12-Q14`

- Q12: 科技公司故意让产品寿命短。
- Q13: 产品要列出 repairability score。
- Q14: 企业应主动减少电子垃圾。

这组是典型 Section B 链条：问题 -> 衡量方式 -> 建议。盲推时，看到“更新产品更酷”“建更多回收设施”这类偏局部或偏跳跃的项，要先降权。

#### Section C：抓研究/概念关系，不抓学术词堆

`2023.06.set1 Q18`

- A 是 `Anxiety has become a serious problem...`，问题词很重，但太泛。
- B 是 `Extreme levels of anxiety can adversely affect cognitive performance`。
- C 是 `may forget things selectively`，过窄。
- D 是 `no direct correlation...`，否定太绝。
- 正确答案是 B。

Section C 常考“发现是什么”。B 把“极端焦虑 -> 认知表现受影响”这条关系说完整，所以比 A 的大问题词、D 的否定词更像答案。

`2024.12.set2 Q22-Q25`

- Q22 先问为什么气味能触发情绪记忆。
- Q23 再问气味分子和大脑区域如何互动。
- Q24 落到 context。
- Q25 总结气味记忆会不准确、会变化。

这组是 Section C 的标准概念链：现象 -> 机制 -> 条件 -> 总结。后题没听到时，不要选孤立地名、孤立例子或只有学术壳的选项，要找能接住这条概念链的项。

### 9. 考场上怎么把这套用起来

1. 先看题型，不先押字母。
2. 看到特别长、特别像解释稿的选项，先怀疑。
3. 看到特别短、但已经把动作关系说完整的选项，先留着。
4. 看到同维度的正反项，先保留，不要急着排。
5. Section A 先想“说话人怎么推进”；Section B 先想“段落功能”；Section C 先想“研究/概念链”。
6. 如果前一题已经听到了，把它压成 1 到 2 个锚点词，再看后题。
7. 完全没听到时，题号先验只当最后的平局裁判。

### 10. 一句话总结

盲猜不是瞎蒙字母，而是先排掉命题人最爱塞的诱饵，再拿前后题的逻辑链去做最后选择。
